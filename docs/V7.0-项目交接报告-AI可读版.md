## 零、快速上手指南（给下一轮 AI 的第一条指令）

**如果你是下一轮 AI 会话，请按以下步骤开始工作：**

1. **了解项目**：阅读本文第一章（项目概要）和第三章（核心架构）
2. **了解风险**：阅读第七章（关键已知问题），特别是 Chaquopy 迭代器问题
3. **定位代码**：参考第十章（关键文件速查），找到需要修改的文件

6. **开始开发**：从第十一章（下一步建议）中选择一个任务开始

- 修改 Python 代码时，必须同步更新 `src/` 和 `android/app/src/main/python/src/` 两处

## 七、⚠ 关键已知问题（必读）

### 1. Chaquopy 17.0.0 迭代器兼容性问题（核心！P0）

**错误摘要**：
- `com.chaquo.python.PyObject$1$1$1 cannot be cast to java.util.Map`
- `Value ... of type java.lang.String cannot be converted to JSONObject`

**根本原因**：Chaquopy 17.0.0 迭代 Python 生成器时，`item.toString()` 返回 Java 对象引用（如 `PyObject$1$1$1@cc54ff3`）而非真实字符串内容。`.toJava()` 方法不存在，`as Map` 转换失败。

**临时解决方案**（已实施）：
- `_core.py`：`chat_stream()` 从 `yield` 生成器改为 `return results` 返回列表
- `MainActivity.kt`：`for (item in generator)` 中改为 `JSONObject(item.toString())` 解析

**影响**：流式对话实际上是一次性返回所有 token 后再逐条显示，**失去了真正的逐 token 流式效果**。用户需等待完整回复后才能看到任何文字。

**恢复真正流式的方案**：
1. 升级 Chaquopy 到更新版本，测试生成器迭代是否修复
2. 研究 Chaquopy 源码中 `PyObject.__iter__` 的正确取值方法
3. 如无法修复，可考虑将流式输出改为 HTTP Server-Sent Events（在设备上开本地 HTTP 服务）

- `SettingsDetailActivity.kt` — ~1,600行（世界书CRUD+审核UI+API Key+记忆设置+主动消息设置+对话设置全在此，应拆分）
- `MainActivity.kt` — ~600行（聊天+持久化+通知+导出，合理范围内）
- `MemoryManageActivity.kt` — ~400行（可接受）

### 3. 代码重复：两套 Python 源码

| 位置 | 文件数 | 用途 |
