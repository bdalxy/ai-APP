## 十四、下一步计划：T6.1 流式输出 (SSE)

### 14.1 目标
将当前"等待完整回复后一次性显示"改为"逐 token 流式输出"，显著提升对话体验。

### 14.2 任务拆解
1. **后端** ([@backend-dev])：DeepSeekClient 新增 `chat_stream()` 方法，使用 `stream=True`，yield 每个 delta
2. **桥接** ([@backend-dev])：chat_bridge 新增 `chat_stream()` 接口，返回 Python 迭代器
3. **前端** ([@frontend-dev])：Kotlin 端通过 Chaquopy 迭代器逐 token 读取
4. **前端** ([@frontend-dev])：RecyclerView 逐 token 更新最后一条消息，实现打字机效果
5. **前端** ([@frontend-dev])：发送中禁用输入框，流结束后恢复

### 14.3 验收标准
- 发送消息后，AI 回复逐字显示
- 流输出期间输入框不可用
- 流正常结束或出错后输入框恢复
- 网络中断时优雅降级，显示已接收的部分内容
