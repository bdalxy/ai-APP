## 一、现状分析

### 1.1 现有设计资产盘点

| 资产类型 | 当前状态 | 评价 |
|----------|----------|------|
| 主题名称 | "往世乐土" (Elysian Realm) | 有明确的二次元世界观锚点 |
| 主色调 | 爱莉希雅粉紫 (#FFB7C5 / #9B59B6) | 温暖、辨识度高 |
| 背景 | 暗紫渐变 (#0F0A1A -> #251838) | 深邃沉浸感好 |
| 毛玻璃 | 半透明白色卡片 (#1DFFFFFF + 1dp 描边) | 现代感强 |
| 粒子动画 | 粉/紫/白半透明粒子漂浮 | 增加灵动感 |
| 聊天气泡 | AI: 粉色柔光 / User: 紫粉渐变 | 区分度好 |
| 圆角体系 | 气泡 16dp / 输入框 24dp / 卡片 12dp | 已有层次感 |
| 应用图标 | 紫色底 + 白色聊天气泡 + 省略号 | 辨识度一般 |

### 1.2 现有设计的优势

- **色彩体系完整**：粉紫主色调 + 暗紫背景已经建立了明确的情感基调
- **二次元文化锚点明确**：Elysia/往世乐土主题具有辨识度
- **现代设计语言**：玻璃拟态 + 渐变 + 粒子动画符合 2024-2026 年设计趋势
- **深色模式优先**：项目从一开始就设计为深色主题

### 1.3 需要改进的方面

- **应用图标辨识度不足**：当前图标仅为聊天气泡 + 三个点，在桌面上容易与普通聊天应用混淆
- **浅色模式缺失**：仅有深色模式，限制了使用场景
- **字体未定义**：未指定品牌字体，依赖系统默认
- **动画缺乏体系化**：粒子动画独立存在，缺少整体的动效语言
- **Material 3 适配不完整**：当前主题基于 Material Components（M2），可升级至 Material 3

## 二、设计战略

### 2.1 品牌定位

```
一句话定位：温暖、有灵魂的 AI 陪伴者
```

### 2.2 核心设计关键词

| 关键词 | 解释 | 设计表达 |
|--------|------|----------|
| **温暖** | AI 不是冰冷的工具，是有温度的陪伴 | 暖色系、柔光、柔和的圆角 |
| **灵动** | 角色"活"在手机里，有生命感 | 粒子动画、微动效、呼吸感 |
| **二次元** | ACG 文化认同，虚拟角色的魅力 | 日系配色、角色头像、插画感 |
| **沉浸** | 让用户忘记这是"软件"，进入角色世界 | 深色背景、低干扰、全屏体验 |
| **现代** | 符合当下移动端设计审美 | 玻璃拟态、Material 3、流畅动效 |

### 2.3 设计原则

1. **角色优先**：UI 是舞台，角色是主角。界面不应喧宾夺主
2. **情感化设计**：每一个交互反馈都应有温度（如打字动画、微表情）
3. **渐进式复杂度**：默认简洁，高级功能（记忆管理、设置）深度隐藏
4. **一致性**：颜色、圆角、间距、动效全应用统一规范
5. **可访问性**：确保文字对比度达标（WCAG AA），支持系统字体缩放

## 三、APP 图标设计方案

### 设计要求

- 形状：Adaptive Icon（Android 8.0+），支持不同厂商的遮罩形状
- 尺寸：前景 108dp x 108dp，安全区域 66dp 内圈
- 辨识度：在 48dp 小图标下仍可识别
- 风格：符合二次元/AI 陪伴的定位
- 不得使用复杂的文字（图标尺寸下不可读）

### 方案 A：「星光之绊」-- 星形 + 聊天元素

**概念**：一颗闪耀的粉紫色星星，星形中央嵌入微小的聊天气泡剪影，象征 AI 角色如星辰般陪伴用户。

**视觉描述**：

```
│         ╲      ╱            │
│    ╲     ╲    ╱     ╱      │
│     ╲     ╲  ╱     ╱       │
│      ╲     ╲╱     ╱        │
│       ╲    ╱╲    ╱         │
│        ╲  ╱  ╲  ╱    ← 星形主体  │
│         ╲╱ 💬 ╲╱     ← 中心聊天气泡 │
│         ╱╲    ╱╲              │
│        ╱  ╲  ╱  ╲             │
│       ╱    ╲╱    ╲            │
│      ╱     ╱╲     ╲           │
│     ╱     ╱  ╲     ╲          │
```

**颜色方案**：
- 星形主体：线性渐变 `#FFB7C5`(粉) -> `#9B59B6`(紫) -> `#7B2D8B`(深紫)
- 中心气泡：纯白 `#FFFFFF`，内部配粉色微小心形 `#FFB7C5`
- 背景层：深紫渐变 `#0F0A1A` -> `#1A1028`

**象征意义**：
- 星形 = "你生命中的一颗星"（陪伴隐喻）
- 聊天气泡 = 核心功能（对话）
- 心形细节 = 温暖、情感连接

**辨识度评估**：高。星形轮廓在方形/圆形/圆角矩形遮罩下均能识别。

**技术实现**：XML Vector Drawable (前景) + 渐变背景层

**概念**：一个柔和的对话框气泡，但左边的小尾巴被设计成心形曲线的延伸，气泡内部有微笑弧线。

```
│           ╲    ♡              │
│            ╲                │
│             ╲╲             │
│                              │

```

**颜色方案**：
- 气泡主体：暖粉色渐变 `#FFD0D9` -> `#FFB7C5`
- 描边：紫色 `#9B59B6`，2dp 粗细
- 心形尾巴：渐变紫 `#C77DFF` -> `#9B59B6`
- 微笑弧线：深紫 `#7B2D8B`
- 背景层：暖色渐变 `#FFE4E9` -> `#FFD0D9`（浅色）/ `#1A1028` -> `#251838`（深色）

**象征意义**：
- 对话框 = 聊天应用的核心动作
- 心形尾巴 = "每次对话都是一次心的交流"
- 微笑弧线 = 温暖、愉悦的陪伴体验

**辨识度评估**：中高。在圆形遮罩下心形尾巴可能被裁剪，需确保主体在安全区域内。

**技术实现**：XML Vector Drawable (前景) + 渐变背景层，尾巴使用贝塞尔曲线 Path。

### 方案 C：「伴月」-- 月牙 + 剪影（推荐备用）

**概念**：一轮弯月怀抱着一颗小星星，月亮象征夜晚的陪伴，星星象征 AI 角色。极简风格，高辨识度。

**颜色方案**：
- 月牙：白色/浅粉 `#FFF0F3`
- 星星：粉色 `#FFB7C5` 发光效果
- 背景：深紫 `#0F0A1A`

**优势**：极简、辨识度极高、不依赖文字、不受遮罩形状影响。  
**劣势**：与聊天/对话的直接关联性较弱。

## 五、字体建议

### 5.1 现有问题

当前项目未定义品牌字体，完全依赖 Android 系统默认字体（Roboto / Noto Sans），缺乏品牌个性。

### 5.2 候选字体方案

#### 方案 A：Noto Sans SC（推荐）

| 属性 | 详情 |
|------|------|
| 名称 | Noto Sans SC (思源黑体) |
| 来源 | Google Fonts，Apache 2.0 开源 |
| 优势 | Android 10+ 系统内置，无需额外下载；简体中文优化；与 Material Design 天然契合 |
| 劣势 | 过于通用，缺乏独特性 |
| 适用 | 正文字、系统 UI 文本 |
| 字号/行高 | Body: 15sp/1.5 | Caption: 12sp/1.3 |

#### 方案 B：Zen Maru Gothic（推荐标题用）

| 属性 | 详情 |
|------|------|
| 名称 | Zen Maru Gothic (禅圆体) |
| 来源 | Google Fonts，OFL 开源 |
| 优势 | 圆体设计，温暖可爱，完美契合二次元/陪伴感；Japanese/中文支持好 |
| 劣势 | 需打包字体文件（约 3-5MB）；全应用使用可能过于"可爱" |
| 适用 | 角色名、对话气泡、标题文字 |
| 字号/行高 | Title: 18sp/1.4 | Subtitle: 14sp/1.4 |

#### 方案 C：M PLUS Rounded 1c

| 属性 | 详情 |
|------|------|
| 名称 | M PLUS Rounded 1c |
| 来源 | Google Fonts，OFL 开源 |
| 优势 | 日系圆体，ACG 氛围感强，比 Zen Maru 更克制 |
| 劣势 | 中文支持为日本汉字，部分简体字可能缺字 |
| 适用 | 仅日语/英文场景或角色名装饰 |

### 5.3 推荐方案

**混合使用方案**（平衡独特性与性能）：

| 用途 | 字体 | 字重 | 字号 |
|------|------|------|------|
| **角色名 / 标题** | Zen Maru Gothic | Bold (700) | 20-24sp |
| **对话气泡正文** | Noto Sans SC | Regular (400) | 15sp |
| **系统 UI 文字** | Noto Sans SC | Regular / Medium | 12-15sp |
| **时间戳 / 辅助信息** | Noto Sans SC | Regular (400) | 11sp |
| **设置项标题** | Noto Sans SC | Medium (500) | 15sp |
| **设置项说明** | Noto Sans SC | Regular (400) | 13sp |

**实施建议**：
1. Zen Maru Gothic 仅以 `res/font/` 方式打包使用，限制在标题和角色名场景
2. 通过 `build.gradle.kts` 配置 `downloadable fonts` 实现按需下载，避免 APK 体积膨胀
3. 提供系统字体回退：`fontFamily="zen_maru_gothic, sans-serif"`

### 5.4 字体层级

```
H1: Zen Maru Gothic Bold 24sp     — 角色卡片名称
H2: Zen Maru Gothic Bold 20sp     — 页面标题
H3: Noto Sans SC Medium 18sp     — 对话框标题
Body: Noto Sans SC Regular 15sp  — 对话正文
Body2: Noto Sans SC Regular 13sp — 设置项说明
Caption: Noto Sans SC Regular 11sp — 时间戳
```

## 六、UI 风格规范

### 6.1 圆角体系

基于现有设计进行标准化：

| 元素 | 圆角 | 说明 |
|------|:---:|------|
| 大卡片 (设置卡片、角色卡片) | **16dp** | 从现有 12dp 微调，更柔和 |
| 对话气泡 | **16dp** | 保持现有，与卡片统一 |
| 输入框 | **24dp** | 保持现有，高度圆角强化亲和力 |
| 按钮 (FAB、发送按钮) | **24dp** | 与输入框统一 |
| 小标签、Chip | **8dp** | 略圆润 |
| 对话框/底部弹窗 | **20dp** (顶部) | 顶部圆角，底部直角 |
| 头像 | **50% (圆形)** | 角色头像用圆形 |

### 6.2 阴影/深度系统

Material 3 的 tonal elevation 方案更适合深色主题：

| 层级 | 阴影描述 | 用途 |
|:---:|------|------|
| 0dp | 无阴影 | 页面背景 |
| 1dp | `rgba(0,0,0,0.15)` blur 4px | 卡片、列表项 |
| 2dp | `rgba(0,0,0,0.20)` blur 8px | FAB、底部弹窗 |
| 3dp | `rgba(0,0,0,0.25)` blur 16px | 对话框 |

**深色模式替代方案**：在深色背景下，阴影不明显，改用**亮度提升** + **微弱描边**来表现层级：
- 表面提升：从 `bg_mid` 到 `bg_light`
- 描边：`glass_border` (半透明白)

### 6.3 渐变规范

| 渐变 | 颜色 | 角度 | 用途 |
|------|------|:---:|------|
| 主背景 | `bg_dark` -> `bg_mid` -> `bg_light` | 180deg(从上到下) | 全页面背景 |
| 用户气泡 | `#C77DFF` -> `#9B59B6` | 135deg | 用户消息气泡 |
| FAB 按钮 | `#FFB7C5` -> `#E0608A` | 45deg | 主要操作按钮 |
| 顶栏淡入 | 实色 -> 透明 | 180deg | 顶栏与内容过渡 |

### 6.4 玻璃拟态规范

当前毛玻璃效果已在使用，标准化如下：

```
背景模糊: backdrop-blur 20px (实现时用 RenderScript 或 RenderEffect)
背景色:  rgba(255, 255, 255, 0.11)  — 深色模式
         rgba(255, 255, 255, 0.60)  — 浅色模式
边框:    rgba(255, 255, 255, 0.18)  — 1dp
圆角:    16dp

适用元素: 顶栏、底栏、设置卡片、角色信息浮层
不适用:   对话气泡、输入框（已有独立样式）
```

### 6.5 动效语言

| 场景 | 动效类型 | 时长 | 缓动 |
|------|----------|:---:|------|
| 页面转场 | 水平滑动 + 淡入 | 300ms | Decelerate |
| 消息出现 | 从下弹出 + 淡入 | 250ms | Overshoot |
| AI 打字中 | 三点跳跃动画（已有） | 循环 1.2s | Linear |
| 角色头像呼吸 | 缩放 100%->105%->100% | 循环 3s | EaseInOut |
| FAB 展开/收起 | 旋转 + 缩放 | 200ms | Decelerate |
| 设置项切换 | 交叉淡入淡出 | 200ms | Linear |
| 粒子背景 | 匀速漂浮（已有） | 持续 | Linear |

**微交互建议**：
- 发送按钮：按下时缩小到 95%，松开弹回 + 轻微脉冲
- 长按消息：触觉反馈 + 消息微微上浮
- 下拉刷新：角色图标旋转 + 弹性回弹

## 七、深色/浅色模式适配

### 7.1 切换策略

- 默认跟随系统（`AppCompatDelegate.MODE_NIGHT_FOLLOW_SYSTEM`）
- 设置面板提供手动切换：跟随系统 / 始终深色 / 始终浅色
- 切换时使用平滑过渡动画（300ms 交叉淡入淡出）

### 7.2 关键差异处理

| 元素 | 深色模式 | 浅色模式 |
|------|----------|----------|
| 背景色 | `#0F0A1A` 深紫 | `#FFF5F7` 暖粉白 |
| 毛玻璃 | 透明黑底 | 透明白底 |
| 粒子动画 | 保留（降低透明度到 60%） | 改为暖色（粉色/金色）或关闭 |
| 气泡对比度 | AI 气泡白色足够 | AI 气泡粉色更柔和 |
| 状态栏 | 白色图标（默认） | 深色图标（`lightStatusBar=true`） |
| 导航栏 | 深色底 + 白色按钮 | 白色底 + 深色按钮 |
| 阴影 | 不可见，用亮度提升替代 | 使用柔和阴影 |

### 7.3 色彩映射表示例

```
深色 → 浅色映射:
  bg_dark (#0F0A1A)   → light_bg (#FFF5F7)
  bg_mid  (#1A1028)   → light_surface (#FFFFFF)
  bg_light (#251838)  → light_surface_variant (#F5EEF2)
  text_primary        → light_on_bg (#3D2A35)
  text_secondary      → light_on_bg_secondary (#6E5A65)
  glass_bg            → light_glass_bg (#99FFFFFF)
  glass_border        → light_glass_border (#40E0D4DB)
```
