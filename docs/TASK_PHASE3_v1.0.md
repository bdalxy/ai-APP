## 任务详情

### T1: 统一配置源策略（P0 — 基础任务）

| 属性 | 值 |
|------|-----|
| 文件 | `AppConfig.kt`, `settings.py`, `chat_bridge/_state.py` |
| 前置 | 无 |
| 后置 | T2, T3, T4 依赖此任务 |

**问题：** Kotlin `AppConfig`(SharedPreferences) 和 Python `settings.py`(环境变量) 双重配置源，`set_api_key()` 已同步但角色卡参数（名称/性格/风格/背景）没有同步机制。

**目标：** 
1. Python 侧新增 `sync_from_appconfig()` 方法，接收 Kotlin 传入的角色卡参数
2. Kotlin 端 `init()` 时将 AppConfig 的参数统一传给 Python
3. `apply_params()` 返回实际生效的完整参数，Kotlin 端记录确认

**测试标准：**
- `set_character_card()` 修改角色后，`get_character_card()` 返回一致值
- `apply_params()` 返回的 params 与 `AppConfig` 中存储的一致
- 重启 APP 后角色卡参数正确恢复

---

### T2: 记忆注入集成到 chat() 流程（P0）

| 属性 | 值 |
|------|-----|
| 文件 | `chat_bridge/_core.py`, `role_player.py` |
| 前置 | T1（确保配置正确） |
| 后置 | 无 |

**问题：** 当前需要手动调用 `inject_memories()` 才能让 AI 感知记忆，记忆未自动注入到对话上下文。

**目标：**
1. `chat()` 方法中，在发送 API 请求前自动检索并注入相关记忆
2. 缓存上次注入的记忆数量，每 N 轮（默认3轮）更新一次，避免过度检索

**测试标准：**
- 发送消息后，日志显示记忆自动注入
- 连续发送消息时，每3轮触发一次记忆更新
- 记忆数量为0时不报错，正常对话

---

### T3: 加密方案升级 XOR → Fernet（P1）

| 属性 | 值 |
|------|-----|
| 文件 | `vector_store.py` |
| 前置 | T1（加密密钥管理需要统一配置） |
| 后置 | 无 |

**问题：** 当前使用 XOR 加密记忆内容，安全性不足。

**目标：**
1. 生成 Fernet 密钥替代 XOR 密钥
2. 兼容旧数据：读取时尝试 Fernet 解密，失败则尝试 XOR 解密，自动迁移
3. 新增 `encrypted` 标记字段防止解密静默失败

**测试标准：**
- 新存储的记忆使用 Fernet 加密
- 旧 XOR 加密的记忆可正常读取并自动迁移
- 解密失败时有明确日志告警

### T4: 深色模式适配（P1）

| 属性 | 值 |
|------|-----|
| 文件 | `values-night/themes.xml`, `values-night/colors.xml`, 各布局文件 |
| 前置 | T1（主题色从统一配置读取） |
| 后置 | T5 |

**问题：** `forceDarkAllowed=false`，无 `values-night/` 目录，深色模式下颜色错乱。

**目标：**
1. 创建 `values-night/themes.xml` 和 `values-night/colors.xml`
2. 定义深色主题色板（暗紫/暗粉/暗灰背景色系）
3. 确保所有硬编码颜色引用 @color 资源

**测试标准：**
- 切换系统深色模式后，APP 自动切换主题
- 聊天气泡、按钮、输入框颜色适配深色背景
- 文字对比度在深色模式下可读

### T5: 横屏 + 平板适配（P1）

| 属性 | 值 |
|------|-----|
| 文件 | `layout-land/`, `values-sw600dp/dimens.xml`, `values-sw840dp/dimens.xml` |
| 前置 | T4（需要深色资源配合） |
| 后置 | T6 |

**问题：** 无横屏布局，气泡宽度硬编码 280dp，平板显示不佳。

**目标：**
1. 创建 `layout-land/` 横屏布局目录，优化聊天气泡占比
2. 完善 `values-sw600dp/dimens.xml`（小平板）和 `values-sw840dp/dimens.xml`（大平板）
3. 气泡最大宽度改为百分比约束（横屏 60%，竖屏 75%）

**测试标准：**
- 横屏时聊天气泡不溢出，布局合理
- 平板上（sw600dp+）气泡宽度适当增加
- 横竖屏切换不丢失输入内容

### T6: 空状态/加载中/错误三态统一（P2）

| 属性 | 值 |
|------|-----|
| 文件 | 各 Activity/Adapter 布局文件 |
| 前置 | T5（布局适配稳定后执行） |
| 后置 | T7 |

**问题：** 各页面空状态处理不一致，有的用 Toast，有的无提示，缺少统一加载/错误态。

**目标：**
1. 创建 `EmptyStateView` / `LoadingView` / `ErrorView` 三个通用组件
2. 记忆管理页：无记忆时显示空状态引导，加载中显示进度，错误显示重试
3. 角色卡页：空列表显示创建引导

**测试标准：**
- 首次打开记忆管理页显示空状态提示
- 记忆加载中显示加载动画
- 加载失败显示错误 + 重试按钮

### T7: DiffUtil / ViewBinding 迁移（P2）

| 属性 | 值 |
|------|-----|
| 文件 | `ChatAdapter.kt`, `MemoryAdapter.kt`, `CharacterAdapter.kt` |
| 前置 | T6（布局稳定后迁移 DiffUtil） |
| 后置 | 无 |

**问题：** RecyclerView 无 DiffUtil，每次 `notifyDataSetChanged()` 导致全量刷新闪烁。

**目标：**
1. ChatAdapter、MemoryAdapter、CharacterAdapter 实现 DiffUtil.ItemCallback
2. 使用 `ListAdapter` 替代 `RecyclerView.Adapter` + 手动列表管理
3. 考虑 ViewBinding 替代 findViewById（可选，视复杂度决定）

**测试标准：**
- 新增消息时不闪烁旧消息
- 记忆列表翻页加载无跳动
- 角色卡列表切换流畅
