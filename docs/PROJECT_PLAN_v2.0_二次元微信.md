## 十五、风险与注意事项

### 15.1 风险清单

| 风险 | 影响 | 概率 | 缓解措施 |
|------|:---:|:---:|------|
| Chaquopy 兼容性问题（新 API） | 高 | 中 | 新功能优先使用纯 Kotlin 实现 |
| Android SpeechRecognizer 设备兼容性 | 中 | 高 | 提供备选方案（Whisper API） |
| Android TTS 中文语音包缺失 | 中 | 中 | 检测引擎可用性，提供下载引导 |
| DuckDuckGo API 被墙 | 高 | 中 | 提供 Bing API 备选 |
| 记忆系统增强导致性能下降 | 中 | 低 | 分批处理，异步执行 |
| 阶段间依赖导致延期 | 中 | 中 | 并行开发独立模块 |
| 酒馆格式兼容性变化 | 低 | 低 | 锁定 spec v2.0 |

### 15.2 注意事项

1. **每阶段完成后必须提交 GitHub**：确保版本可追溯
2. **每阶段完成后必须人工审核**：测试合格标准见各阶段验收标准
3. **新需求先查看需求文件和任务文件**：避免重复开发
4. **不随意引入新技术栈**：严格基于现有架构
5. **所有数据本地存储**：不依赖云服务（除 DeepSeek API 外）
6. **Android 原生开发**：不是 Web 应用，不是微信小程序
7. **测试优先**：每个阶段完成后在真机上验证

### 15.3 协作模式

```
项目经理 (@project-manager) → 分析需求、拆分任务、分配智能体
```
