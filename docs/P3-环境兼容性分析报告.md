## 二、适配的手机品牌

### Android 不是苹果生态——不需要适配"品牌"，而是适配"CPU 架构"

Android 应用通过 **ABI（CPU 指令集）** 来兼容设备，而不是品牌。当前配置：

```kotlin
ndk {
    abiFilters += listOf("arm64-v8a", "x86_64")
}
```

### 各 ABI 覆盖的设备

| ABI | 覆盖设备 | 品牌举例 |
|-----|---------|---------|
| **arm64-v8a** | 2016年至今几乎所有真机 | 华为、小米、OPPO、vivo、三星、荣耀、一加、真我、传音、Google Pixel、摩托罗拉、中兴、努比亚... |
| **armeabi-v7a** | 2011-2016 年老设备（32位） | 旧款手机，Python 3.12+ 不再支持 |
| **x86_64** | 模拟器 | Android Studio 自带模拟器 |
| **x86** | 旧模拟器（32位） | Python 3.12+ 不再支持 |

### 分析

- **arm64-v8a**：覆盖 99% 以上在售真机，包括所有主流品牌
- **x86_64**：仅用于开发调试时的模拟器，不影响最终用户
- 不包含 armeabi-v7a 和 x86 是因为 Chaquopy 17.0.0 默认 Python 3.10+，Python 3.12+ 已放弃 32 位支持

### 建议

**保持当前配置 `arm64-v8a + x86_64`**，覆盖所有主流品牌真机 + 模拟器调试。

## 三、JDK 版本兼容性问题

### 关键发现

| 组件 | 当前 | 问题 |
|------|------|------|
| Java 路径 | `F:\Java\jdk-17` | 文件夹名是 jdk-17，但实际装的是 **JDK 25** |
| JDK 实际版本 | 25.0.3 (Temurin) | 太新！ |
| Gradle 版本 | 8.9 | JDK 25 需要 Gradle 9.1+ |
| AGP 版本 | 8.7.0 | Gradle 9.x 兼容性未知 |

### Gradle 对 JDK 的兼容性矩阵

| JDK 版本 | 最低 Gradle 版本 |
|----------|-----------------|
| JDK 17 | Gradle 7.3 ✅ |
| JDK 21 | Gradle 8.5 ✅ |
| JDK 22 | Gradle 8.8 ✅ |
| **JDK 25** | **Gradle 9.1.0** ❌ (当前 Gradle 8.9) |

### 结论

**JDK 25 是导致 "Unresolved reference" 和构建失败的根本原因之一。** Gradle 8.9 无法在 JDK 25 上正常运行，导致 Chaquopy 插件加载失败。

### 解决方案

**安装真正的 JDK 17 到 `F:\Java\jdk-17`**（覆盖当前的 JDK 25）。

| 方案 | 优点 | 缺点 |
|------|------|------|
| **装 JDK 17**（推荐） | 最稳定，Android 开发黄金标准，兼容 AGP 8.7.0 + Gradle 8.9 | 需要重新下载安装 |
| 升级 Gradle 到 9.1+ | 不需要换 JDK | AGP 8.7.0 可能不兼容 Gradle 9.x，风险大 |

**推荐：** 从 [Adoptium](https://adoptium.net/) 下载 **JDK 17 (Temurin)** 安装到 `F:\Java\jdk-17`。

## 五、其他需要注意的问题

### 5.1 项目路径包含中文

```
F:\Trae AI\开发\android\
```

虽然已通过 `android.overridePathCheck=true` 绕过 Gradle 检查，但中文路径可能导致其他问题：
- 某些 NDK 工具链可能不支持中文路径
- 部分三方库的构建脚本可能崩溃
- 排查问题时难以区分是路径问题还是代码问题

**建议：** 长期考虑迁移到纯英文路径，如 `F:\projects\ai-chat-app\`。短期可继续使用，但如遇莫名错误先排查路径。

### 5.2 Python 版本选择

当前配置 Chaquopy 使用 Python 3.10（默认版本）。

Chaquopy 17.0.0 支持的 Python 版本：
- 3.10（默认）、3.11、3.12、3.13、3.14

选择 3.10 的理由：
- 这是 Chaquopy 17 的默认版本
- 包兼容性最好
- 不需要在构建机上安装 Python（Chaquopy 内置）
- 项目 Python 代码原本就是 3.10

### 5.3 Python 依赖包

当前项目 Python 代码依赖：
- `openai`（DeepSeek API 调用）
- `loguru`（日志，已做 Android 兼容处理）
- `chromadb`（向量存储）
- 其他标准库

这些依赖需要在 `chaquopy { defaultConfig { pip { } } }` 中声明。Chaquopy 会自动下载 Android 预编译的 wheel 包。如果某个包没有 Android wheel，构建会失败，需要找替代方案。

### 5.4 Android Studio 中文语言包

你说"汉语包用不了我看不懂"。Android Studio 的中文语言包是 IDE 插件，需要从 Marketplace 安装。但建议：
- 使用英文版 Android Studio 更稳定
- 错误信息用英文更容易搜索解决方案
- 如确需中文，可在 **File → Settings → Plugins → Marketplace** 搜索 "Chinese" 安装

### 5.5 磁盘空间

- Android SDK 通常占用 5-15 GB
- Gradle 缓存 (`~/.gradle/caches`) 可能占用 2-5 GB
- 建议定期清理 Gradle 缓存和不需要的 SDK 版本
