## 五、技术分析

### 问题本质

`Unresolved reference: python` 和 `buildPython` 说明 **`com.chaquo.python` Gradle 插件未被加载到 classpath**。

`python { }` DSL 块由 `com.chaquo.python` Gradle 插件提供。如果该插件无法解析/下载，此 DSL 块就不存在，编译报错。

### 可能原因

1. **Chaquopy Maven 仓库无法访问** — `https://chaquo.com/maven` 可能被墙或网络不通
2. **Chaquopy 插件版本不兼容** — `15.0.1` 可能与 AGP 8.7.0 或 Gradle 8.9 不兼容
3. **Chaquopy IDE 插件缺失** — 某些版本的 Chaquopy 需要 IDE 插件提供 DSL 支持
4. **Gradle 缓存问题** — 之前下载失败后的缓存损坏

### 疑问

- Chaquopy 在 AGP 8.7.0 + Gradle 8.9 上的最新兼容版本是什么？
- 是否需要同时安装 Chaquopy IDE 插件（在 Android Studio Marketplace 中找不到）？
- 是否可以通过离线方式（本地 jar/aar）引用 Chaquopy？
