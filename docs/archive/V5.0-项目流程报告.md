## 一、项目概述

**AI Companion** 是一款运行在 Android 手机上的 AI 角色扮演聊天应用。用户与预设的 AI 角色（"小美"）进行自然对话，AI 会记住用户说过的重要信息，下次聊天时能主动回忆。

### 核心体验三角

```
          聊天质量
          (角色扮演)
           /    \
          /      \
   记忆系统 ------ 主动消息
  (像真人记忆)    (AI主动发起)
```

### 技术栈一览

| 层级 | 技术 | 用途 |
|------|------|------|
| Android UI | Kotlin + 原生 View (XML) | 聊天界面、设置面板、记忆管理 |
| Python 运行时 | Chaquopy 17.0.0 | 在 APK 内嵌入 Python 3.10 运行时 |
| AI 接口 | DeepSeek API (OpenAI 兼容格式) | Chat（对话）+ Embedding（向量化） |
| 记忆存储 | SQLite (WAL 模式) | 持久化向量 + 元数据 + 倒排索引 |
| 构建 | Gradle 8.x + AGP 8.x | APK 构建（仅 arm64-v8a） |
| 开发语言 | Kotlin 2.0.20 + Python 3.10 | 前后端分离 |

## 三、各阶段开发内容

### P0: 核心决策（2026-06-04）

**关键决策**：

1. **产品形态**：纯 Android APK（自用为主，少数人测试）
2. **技术路线**：Python 嵌入 Android → 选择 **Chaquopy** 方案
3. **AI 后端**：DeepSeek API（保留本地 LLM 接口）
4. **三大核心体验**：聊天质量、记忆系统、主动消息
5. **架构要求**：可维护、可扩展

**决策过程**：评估了 5 种方案（Termux/服务器/Kivy/Chaquopy/PWA），最终选择 Chaquopy 是因为它能让 Python 代码直接打包进 APK，用户无需安装额外运行环境，同时保留原生 Android UI。

### P4: 记忆系统（2026-06-08 ~ 06-09）

这是迄今为止投入最大、审核最严格的阶段，经历了 **6 轮交叉审核/修复**。

#### P4-1: 交叉审核（2026-06-08）

**方式**：三个智能体（架构师 / 后端开发 / QA 测试）并行交叉审核 `src/memory/` 全部 5 个模块。

**核心结论**：
- 底层组件质量良好
- **最大问题**：缺少编排层将组件串联，优秀模块在 P3 中完全没有被调用

**发现的缺陷**（13 项）：
- P0（7 项）：TOCTOU 竞态、close() 无防护、衰减重复计算、余弦双重计算、去重缺语义、错误静默吞没、WAL 无 checkpoint
- P1（6 项）：embed 无缓存、LLM 无节流、全量回退 OOM、WAL 兼容性、游标泄漏、去重复杂度

#### P4-2: 缺陷修复（2026-06-08）

修复全部 7 个 P0 缺陷，涉及 5 个文件，约 110 行改动：
- `vector_store.py`：close 防护、TOCTOU 修复、错误传播、WAL checkpoint、search 返回带分数结果
- `decay.py`：get_weight 优先使用 entry.decay_factor
- `retriever.py`：消除余弦相似度双重计算
- `extractor.py`：去重优先语义向量
- `exporter.py`：移除未实现的 batch_size

#### P4-3: 编排层开发（2026-06-08）

**新增文件**：`orchestrator.py`（255 行）

**核心方法**：
| 方法 | 流程 | 返回值 |
|------|------|--------|
| `remember()` | extract(rule模式) → embed → add | 存储条数 |
| `recall()` | retrieve(含衰减加权) → 提取纯文本 | 记忆文本列表 |
| `get_stats()` | 统计总数/各类型/轮次 | 统计字典 |

**chat_bridge.py 新增 4 个接口**：
- `init_memory(db_path)` — 初始化记忆系统
- `inject_memories(query_text)` — 检索记忆并注入 System Prompt
- `get_memory_stats()` — 获取统计信息
- `remember_turn(...)` — 手动补录记忆

**一键对话流程打通**：
```
用户输入 → inject_memories() → recall() → 注入 System Prompt
         → chat() → 角色扮演 → AI 回复
         → _auto_remember() → extract() → embed() → store()
```

#### P4-7: 记忆系统初始化失败交叉审核（2026-06-09）

**发现根因**：db_path 路径拼接错误。

```
Kotlin: filesDir.absolutePath + "/memory.db"      → /data/.../files/memory.db
Python: os.path.join(db_path, "memories.db")       → /data/.../files/memory.db/memories.db ❌
```

`os.path.join` 将 `memory.db` 当作目录，追加 `memories.db` 后路径无效。

**为何问题隐蔽**：层层吞错导致 — Python try/except → 返回 error JSON → Kotlin 不检查 status → 打印假成功 → 功能静默失效。

**发现全部 Bug**（12 项，按严重度）：

| 级别 | Bug | 描述 |
|:---:|-----|------|
| P0 | db_path 路径拼接错误 | 记忆系统完全无法初始化 |
| P0 | initMemory() 不检查 status | 错误时打印"初始化成功" |
| P0 | API异常时 memories 不清空 | 幽灵记忆残留 |
| P1 | _auto_remember() 同步阻塞 | 对话延迟增加 |
| P1 | initMemory() 独立协程竞态 | 记忆/对话初始化不同步 |
| P1 | _auto_remember() 静默吞错 | 存储失败完全不可见 |
| P2 | embed API 逐条调用未批量 | 浪费 token 和时间 |
| P2 | extractJsonInt 无法区分 -1 | 字段不存在/解析失败/真值-1 无法区分 |
| P2 | 无 embedding 缓存 | 相同文本重复调用 |
| P3 | orchestrator 死代码 | 外层 try/except 永不可达 |
| P3 | _turn_counter 非线程安全 | 低风险但规范不足 |
| P3 | _record_access 用 debug 级别 | 生产环境看不到访问记录失败 |

**修复方案**（立即修复 5 项）：
- F1: Kotlin 传目录路径，去掉 `/memory.db`
- F2: initMemory 检查 status 字段
- F3: role_player.chat() 用 try/finally 清空 memories
- F4: 消除 initMemory 独立协程，改为 suspend
- F5: _auto_remember 添加日志

**当前修复状态**：部分修复通过 `V4.2-交叉审核修复报告` 已完成（R5/M1/M2/B1/B2/B3），F1-F5 中 F3 已完成，**F1/F2/F4/F5 待确认是否已实施**。

## 四、决策记录

本项目的所有重要技术决策及其理由：

| # | 决策 | 选择 | 备选方案 | 理由 |
|:---:|------|------|------|------|
| 1 | Python 嵌入方案 | **Chaquopy** | Termux / 服务器 / Kivy / PWA | 唯一成熟方案，可嵌入纯 Python 依赖，用户无需额外安装 |
| 2 | Android UI 方案 | **传统 View (XML)** | Jetpack Compose | 简单、兼容性好，项目规模不大 |
| 3 | AI 后端 | **DeepSeek API** | 本地 LLM / 其他云端 API | 性价比高，中文效果好，OpenAI 兼容格式 |
| 4 | 向量存储 | **SQLite + 手动余弦** | ChromaDB / Pinecone / Milvus | 离线可用，零依赖，Android 原生支持 |
| 5 | 记忆提取 | **规则 + LLM 双模式** | 纯 LLM / 纯规则 | 规则免费快速，LLM 智能准确，按需切换 |
| 6 | 混合检索 | **倒排 → 余弦精排** | 纯倒排 / 纯余弦 / ANN | 平衡准确率和性能 |
| 7 | ABI 过滤 | **仅 arm64-v8a** | 全 ABI | 减小 APK 体积，避免 Chaquopy 17.0.0 bug |
| 8 | 日志系统 | **loguru → 标准logging降级** | 纯 loguru / 纯 logging | Android 环境兼容 |
| 9 | 主动消息 | **P1 手动触发 → P5 后台推送** | 直接做后台推送 | 先验证逻辑，再解决推送通道问题 |
| 10 | 构建方式 | **Gradle + Chaquopy** | Bazel / 手动 | Gradle 是 Android 标准构建 |
| 11 | API Key 注入 | **运行时从 SharedPreferences 传入** | 硬编码 / 打包进 APK | 安全：不嵌入 APK，用户自行配置 |
| 12 | Python 依赖安装 | **手动提取 wheel 包** | Chaquopy pip | 绕过 Chaquopy 17.0.0 的 move_to_common bug |

```
开发者完成代码
    ↓
单智能体自审核（编译/语法检查）
    ↓
交叉审核（2-3 个智能体并行独立审查）
    ↓
对比结论（一致 → 通过 / 分歧 → 讨论）
    ↓
修复 → 构建验证 → 手机安装测试
```
