## 一、项目概述

### 1.1 基本信息

| 属性 | 值 |
|------|-----|
| **项目名称** | AI Companion (AI伴侣) |
| **项目定位** | 基于 Android + Python 的 AI 角色扮演聊天应用 |
| **核心功能** | 用户与 AI 角色"小美"进行自然对话，系统具备长期记忆存储、语义检索和上下文注入功能 |
| **应用包名** | `com.aicompanion.app` |
| **当前版本** | `0.1.0` (versionCode=1) |
| **开发阶段** | P4 (记忆系统编排层已完成，Android UI 集成待验证) |
| **代码总行数** | 约 6000+ 行 (Kotlin ~600 + Python ~4000 + XML ~200 + 测试 ~800) |

### 1.2 核心设计理念

用户可以与预设的 AI 角色（"小美"）进行自然对话。系统会：
1. **记住**用户说过的重要信息（是记忆，而非仅缓存）
2. **检索**与当前对话相关的历史记忆
3. **注入**到 System Prompt 中，让 AI 在对话中自然提及
4. **衰减**陈旧记忆，保持记忆库的质量和相关性

整个系统运行在 Android 手机上，Python 代码通过 Chaquopy 嵌入 APK，无需网络下载运行时。所有数据存储在本地，仅 DeepSeek API 调用需要网络。

#### 3.2.2 Python 核心模块

**通信桥** (`chat_bridge.py` - 352行)
- Kotlin 与 Python 的唯一通信接口，暴露 9 个 `callAttr` 方法
- 持有 `RolePlayer` 和 `MemoryOrchestrator` 全局单例
- 管理 Python 模块的完整生命周期

**API 客户端** (`api_client/deepseek.py` - 520行)
- `DeepSeekClient`: 封装 Chat Completion + Embedding API
- `CostTracker`: Token 消耗和费用（人民币）追踪
- `ChatResponse` / `EmbeddingResponse`: 响应数据结构

**聊天引擎** (`chat_engine/` - 5模块，约1100行)
- `RolePlayer`: 对话主流程（上下文管理→Prompt构建→API调用→回复存储）
- `CardParser` + `Card`: JSON 角色卡解析和验证（4个必填字段）
- `ContextManager`: 基于 deque 的短期上下文，FIFO 自动裁剪
- `PromptBuilder`: System Prompt 组装（角色卡+世界书+记忆+指引）
- `TokenPreset`: 3种预设模式（quality/balanced/economy）

**记忆系统** (`memory/` - 6模块，约2000行)
- `MemoryOrchestrator`: 编排层，串联提取→向量化→存储→检索→衰减完整链路
- `VectorStore`: SQLite 持久化 + 倒排索引 + 余弦相似度检索
- `MemoryRetriever`: 混合检索（Embed API + 倒排预过滤 + 余弦精排 + 衰减加权）
- `MemoryExtractor`: 双模式提取（规则正则 + LLM API），含去重逻辑
- `decay.py`: 指数衰减算法，三类记忆不同半衰期
- `exporter.py`: JSON 导出（对话历史/长期记忆/完整数据）

**配置管理** (`config/` - 2模块，约300行)
- `Settings`: 单例，从 .env 加载 API Key/模型/URL 等 6 项配置
- `UserSettings`: 16 项用户设置 JSON 持久化，原子写入，线程安全

**主动消息** (`proactive/` - 3模块，约700行)
- `ProactiveEngine`: 决策引擎（时段/间隔检查） + 消息生成
- `ProactiveScheduler`: 发送时间记录/状态持久化
- `TopicGenerator`: 话题生成（AI模式 + 规则模板回退）

**工具模块** (`utils/` - 3模块，约350行)
- `retry.py`: @retry 装饰器（指数退避 + 随机抖动 + 3次重试）
- `time_utils.py`: 时间段判断、ISO 8601 格式化、时间差计算
- `logger.py`: loguru 封装（控制台+文件双路输出，Android 兼容降级）

### 5.4 记忆流：存储 → 检索 → 注入完整链路

```
【存储链路（对话后）】
    用户输入 + AI回复
        │
        ▼
    MemoryExtractor.extract(messages, mode="rule")
        │
        ├─ 规则模式 (默认，零API开销)
        │   └─ 正则: "我是..." "我喜欢..." "我住在..." 等7组规则
        │       → MemoryEntry(type="user_fact"|"episodic", content, importance)
        │
        └─ LLM模式 (已实现，MVP未启用)
            └─ DeepSeek API 结构化提取 → JSON解析 → MemoryEntry列表
        │
        ▼
    _deduplicate(new_entries)
        └─ bigram Jaccard相似度 > 0.85 → 跳过
        │
        ▼
    DeepSeekClient.embed(content) → 1536维向量
        │
        ▼
    VectorStore.add(entry)
        ├─ SQLite INSERT (11列)
        ├─ extract_keywords(content) → bigram/trigram
        └─ InvertedIndex.add(id, keywords) → 倒排索引更新


【检索链路（对话前）】
    用户输入 "还记得我喜欢什么吗？"
        │
        ▼
    MemoryOrchestrator.recall(query_text, top_k=3)
        │
        ▼
    MemoryRetriever.retrieve(query_text, top_k=3, apply_decay=True)
        │
        ├─ [1] DeepSeekClient.embed(query_text) → 查询向量
        │
        ├─ [2] VectorStore.search(query_embedding, query_text, top_k*2)
        │   ├─ extract_keywords(query_text) → 查询关键词
        │   ├─ InvertedIndex.search(query_keywords) → 候选ID集
        │   ├─ 若有候选 → _get_by_ids(ids) → 候选条目
        │   └─ 若无候选 → _get_all() → 全量回退
        │
        ├─ [3] 余弦相似度精排
        │   └─ foreach candidate: cosine_similarity(query_vec, entry.embedding)
        │
        ├─ [4] 时间衰减加权 (apply_decay=True)
        │   ├─ calculate_decay(entry)                    → 指数衰减因子
        │   │   └─ e^(-ln(2) * elapsed_days / half_life) + access_boost
        │   └─ combined_score = cosine_sim * 0.6 + decay_weight * 0.4
        │
        ├─ [5] min_similarity过滤 → top_k返回
        │
        └─ _record_access() → 更新 access_count + last_accessed


【注入链路（构建System Prompt时）】
    RolePlayer.inject_memories(["用户喜欢草莓蛋糕", "用户叫小明", "昨天聊了电影"])
        │
        ▼
    PromptBuilder.build_system_prompt(card, memories=[...])
        │
        ├─ [1] 角色卡 (最高优先级)  → card_max_chars=1200
        ├─ [2] 世界书                → world_book_max_chars=400
        ├─ [3] ★ 记忆文本 ★          → memories_max_chars=300
        │   └─ "## 关于对方的已知信息\n以下是你之前了解到的..."
        │       "请在对话中自然地运用..."
        │       "- 用户喜欢草莓蛋糕"
        │       "- 用户叫小明"
        │       "- 昨天聊了电影"
        ├─ [4] 对话指引              → guideline_max_chars=200
        │
        └─ _truncate(完整system_prompt, max_system_prompt_chars)
```

```python
{
    "model": "deepseek-v4-flash",      # 预设默认/用户自选

**响应处理**:
- 成功: 解析 `choices[0].message.content` → `ChatResponse`
- 失败: HTTP状态码映射为 6 种异常类
- 重试: `@retry(max_retries=3, backoff=2.0, jitter=True)` 自动重试超时/服务器错误/限流

**Token 预设对应模型**:
| 预设 | 默认模型 |

| quality | deepseek-v4-pro |
| balanced | deepseek-v4-flash |
| economy | deepseek-v4-flash |

)
```

**数据库配置**:
- `PRAGMA journal_mode=WAL` (写入不阻塞读取)
