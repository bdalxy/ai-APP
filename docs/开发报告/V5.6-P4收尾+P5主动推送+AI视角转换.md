### T4.5 — Embedding缓存LRU

**问题**：相同文本多次调用embedding API，浪费请求配额。

**方案**：LRU缓存，容量500条。

**文件**：`android/app/src/main/python/src/api_client/deepseek.py`

```python
from collections import OrderedDict

class DeepSeekClient:
    _embedding_cache: OrderedDict[str, list[float]] = OrderedDict()
    _EMBEDDING_CACHE_MAX = 500
    
    def get_embedding(self, text: str) -> list[float]:
        # 1. 检查缓存
        if text in self._embedding_cache:
            return self._embedding_cache[text]
        # 2. 调用API
        result = self._call_embedding_api(text)
        # 3. 写入缓存 + LRU淘汰
        if len(self._embedding_cache) >= self._EMBEDDING_CACHE_MAX:
            self._embedding_cache.popitem(last=False)  # 淘汰最旧
        self._embedding_cache[text] = result
        return result
```

## 四、加密密钥持久化

**问题**：重启应用后，之前AES加密的记忆无法解密（密钥丢失导致乱码）。

**修复**：`vector_store.py`

```python
def _init_encryption(self) -> None:
    key = os.getenv("MEMORY_ENCRYPTION_KEY")
    if not key:
        # 尝试从本地文件加载已持久化的密钥
        key_path = os.path.join(self._db_dir, ".encryption_key")
        if os.path.exists(key_path):
            with open(key_path, "r") as f:
                key = f.read().strip()
            self._log.info("[加密] 已加载持久化 XOR 密钥")
        else:
            # 生成新密钥并持久化
            key = secrets.token_hex(16)
            with open(key_path, "w") as f:
                f.write(key)
            self._log.info("[加密] 已生成并持久化新 XOR 密钥")
```

**效果**：重启应用后记忆内容不再显示乱码。

## 六、AI视角转换 ★核心改进

### 问题

用户反馈：记忆系统存储的内容应该是**AI的记忆**，而不是用户的第一人称陈述。

| 用户说 | 原来存储 | 应该存储 |
|--------|---------|---------|
| 我叫李明 | `我叫李明` | `我知道用户叫李明` |
| 我喜欢打篮球 | `我喜欢打篮球` | `我知道用户喜欢打篮球` |
| 我住在上海 | `我住在上海` | `我知道用户住在上海` |
| 我不喜欢吃香菜 | `我不喜欢吃香菜` | `我知道用户不喜欢吃香菜` |
| 我的生日是5月20日 | `我的生日是5月20日` | `我知道用户的生日是5月20日` |

### 实现

**文件**：`android/app/src/main/python/src/memory/extractor.py`

#### 1. 规则提取 — 新增`_convert_to_ai_perspective()`函数

```python
def _convert_to_ai_perspective(user_text: str) -> str:
    """将用户说的内容从第一人称（用户视角）转换为AI视角。"""
    replacements = [
        (r"^我叫\s*(.*)$", r"我知道用户叫\1"),
        (r"^我是\s*(.*)$", r"我知道用户是\1"),
        (r"^我?喜欢\s*(.*)$", r"我知道用户喜欢\1"),
        (r"^我?不喜欢\s*(.*)$", r"我知道用户不喜欢\1"),
        (r"^我?讨厌\s*(.*)$", r"我知道用户讨厌\1"),
        (r"^我?(住在|家在)\s*(.*)$", r"我知道用户\1\2"),
        (r"^我的\s*(\w+)\s*(是|为)\s*(.*)$", r"我知道用户的\1是\3"),
        # ... 更多模式
    ]
    for pattern, repl in replacements:
        new_text = re.sub(pattern, repl, text)
        if new_text != text:
            return new_text
    # 兜底：开头是"我..."统一处理
    if text.startswith("我"):
        return "我知道用户" + text[1:]
    return text
```

#### 2. LLM提取 — Prompt更新

```
记忆视角：所有记忆必须以AI的视角（"我知道用户..."），而不是用户的第一人称视角。
- 用户说"我叫李明" → memory content: "我知道用户叫李明"
- 用户说"我喜欢打篮球" → memory content: "我知道用户喜欢打篮球"
```

**三道防线确保转换**：
1. 规则提取 → `_convert_to_ai_perspective()` 转换
2. LLM提取 → Prompt要求AI视角
3. 未匹配开头也是"我" → 统一兜底转换

### 架构

```
WorkManager(每2h定时) → ProactiveWorker.doWork()
    ↓

    ↓

```

### T5.1 — Python端消息生成

**文件**：`android/app/src/main/python/chat_bridge.py`

**新增方法**：`generate_proactive_message()`

```python
def generate_proactive_message() -> str:
    """生成主动推送消息。返回JSON: {"status":"ok","message":"..."}"""
    # 1. 获取角色名
    card_info = _ctx.player.get_card_info()
    name = card_info.get("nickname", "AI伙伴")
    
    # 2. 获取最近3条记忆
    memories = _ctx.orchestrator.recall_top_memories(3)
    
    # 3. 构建prompt
    prompt = f"""你是{name}。
当前时间：{now_str}
你记得关于用户的信息：
{memory_text}

请以{name}的身份，说一句自然的主动问候（30字以内）。只输出消息。"""
    
    # 4. 调用LLM
    response = _ctx.client.chat(prompt)
    return json.dumps({"status": "ok", "message": response})
```

**文件**：`android/app/src/main/python/src/memory/orchestrator.py`

**新增方法**：`recall_top_memories(top_k=3)`

```python
def recall_top_memories(self, top_k: int = 3) -> list[str]:
    """获取最近最重要的记忆（不基于查询，用于主动推送）。"""
    entries = self._store.list_with_rowid(limit=50)
    # 按 importance * decay_factor 排序
    entries.sort(key=lambda e: e.importance * _decay_factor(e), reverse=True)
    return [e.content for e in entries[:top_k]]
```

### T5.3 — WorkManager定时器

**新增文件**：`ProactiveWorker.kt`

```kotlin
class ProactiveWorker(
    context: Context,
    params: WorkerParameters
) : CoroutineWorker(context, params) {
    
    override suspend fun doWork(): Result {
        return withContext(Dispatchers.IO) {
            val py = Python.getInstance()
            val module = py.getModule("chat_bridge")
            val result = module.callAttr("generate_proactive_message").toString()
            val json = JSONObject(result)
            if (json.optString("status") == "ok") {
                NotificationHelper.showProactiveMessage(
                    applicationContext,
                    "AI伙伴",
                    json.getString("message")
                )
            }
            Result.success()
        }
    }
}
```

**注册**（`AICompanionApp.kt`）：
```kotlin
val request = PeriodicWorkRequestBuilder<ProactiveWorker>(2, TimeUnit.HOURS)
    .setConstraints(
        Constraints.Builder()
            .setRequiredNetworkType(NetworkType.CONNECTED)
            .build()
    )
    .build()

WorkManager.getInstance(this).enqueueUniquePeriodicWork(
    "proactive_push",
    ExistingPeriodicWorkPolicy.KEEP,
    request
)
```
