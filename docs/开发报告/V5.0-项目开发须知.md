## 零、架构理念（最重要）

### 核心理念：骨架化 + 高扩展性

项目目标是**代码精简但不简陋**，让主体成为可扩展的"骨架"，不是"简易版"。

- **不是硬性体积限制** — APK 体积只要合理就行，不追求"一定小于 X MB"
- **性价比优先** — 如果增加体积能带来更好的体验，是值得的
- **理想状态** — 体验好 + 体积小（但不是以牺牲体验为代价去缩减体积）
- **代码精简的真正含义** — 用最少的代码实现清晰的结构，方便后续加功能像搭积木

> 2026-06-10 用户纠正：之前"轻量化"的说法被过度理解为硬性体积要求。实际目标是骨架化架构，保证扩展性。

## 一、项目概述

AI Companion（AI伴侣）是一个 Android 聊天应用，核心特点：
- 用 **Kotlin** 写 Android UI
- 用 **Python** 写 AI 逻辑（对话引擎、记忆系统、主动推送等）
- 通过 **Chaquopy** 将 Python 代码嵌入 APK
- 调用 **DeepSeek API** 驱动对话

## 三、构建命令

构建和安装 APK 用 **PowerShell**：

```powershell
# 1. 构建 debug APK
cd "F:\Trae AI\ai-APP\android"
.\gradlew assembleDebug --warning-mode none

# 2. 安装到手机（手机需USB连接并开启调试）
& "F:\Android\SDK\platform-tools\adb.exe" install -r "F:\Trae AI\ai-APP\android\app\build\outputs\apk\debug\app-debug.apk"

# 3. 构建+安装一键
cd "F:\Trae AI\ai-APP\android" ; .\gradlew assembleDebug --warning-mode none ; & "F:\Android\SDK\platform-tools\adb.exe" install -r "F:\Trae AI\ai-APP\android\app\build\outputs\apk\debug\app-debug.apk"
```
