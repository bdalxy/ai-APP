## 三、MemoryOrchestrator 设计

```
                    ┌─────────────────────┐
                    │ MemoryOrchestrator   │
                    │                     │
                    │  remember()  ────────┼──→ Extractor → embed → VectorStore.add()
                    │  recall()   ────────┼──→ Retriever → VectorStore.search() → texts
                    │  get_stats()        │
                    │  close()            │
                    │                     │
                    │ 持有:               │
                    │  ├─ VectorStore     │
                    │  ├─ MemoryRetriever │
                    │  └─ MemoryExtractor │
                    └─────────────────────┘
```

### 核心方法

| 方法 | 流程 | 返回值 |
|------|------|--------|
| `remember(turn_id, user_msg, ai_reply)` | extract(rule模式) → embed → add | `int` 存储条数 |
| `recall(query_text, top_k=3)` | retrieve(含衰减加权) → 提取纯文本 | `list[str]` |
| `get_stats()` | 统计总数/各类型/轮次 | `dict` |

### 错误处理策略

| 场景 | 行为 |
|------|------|
| 提取失败 | 返回 0，不阻断对话 |
| embed API 不可用 | 记忆跳过向量化但仍入库 |
| 单条存储失败 | 跳过该条，继续处理其他 |
| 检索失败 | 返回空列表，不抛异常 |
| 记忆库为空 | 跳过检索，直接返回空 |

### MVP 设计决策

- **提取模式固定为 `"rule"`**（正则匹配），零 API 开销。LLM 模式留到 P4 后期启用
- **embed 向量化**是唯一会产生 API 费用的环节（每次存储约 ¥0.000001）
- **记忆存储在 chat() 返回后自动执行**，用户无感知

## 四、chat_bridge.py 新增接口

| 函数 | 用途 | 调用时机 |
|------|------|----------|
| `init_memory(db_path)` | 接收 Android filesDir 路径，初始化记忆系统 | APP 启动后 |
| `inject_memories(query_text)` | 检索记忆并注入 System Prompt | 每次 chat() 前 |
| `get_memory_stats()` | 返回记忆总数和各类型统计 | 设置页面展示 |
| `remember_turn(...)` | 手动补录记忆 | 兼容/测试场景 |

### 自动记忆存储

`chat()` 方法在对话返回后自动调用 `_auto_remember()`：
```python
if _orchestrator is not None and reply:
    _auto_remember(user_input, reply)
```
- 生成递增 turn_id：`"turn_{N}_{timestamp}"`
- 所有异常静默处理，不影响对话体验

## 七、已知局限

1. **提取仅规则模式**：正则能覆盖约 60-70% 的常见记忆模式（"我叫/我喜欢/我住在/今天..."），语义较复杂的记忆需后续启用 LLM 提取
2. **无 Android UI**：Kotlin 端尚未添加记忆设置入口（调用 `init_memory` / `get_memory_stats`），需追加 UI 层
3. **无记忆可视化**：用户无法在 APP 内查看已存储的记忆
4. **P1 问题未修复**：embed 缓存、节流、全量回退 OOM 等（见 [V2.3-P1待修复清单](./V2.3-P4-P1待修复问题清单.md)）
