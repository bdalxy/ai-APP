# AI Companion 项目详细报告

> 生成时间：2026-06-09 | 版本：P4-beta | 语言：Kotlin + Python

---

## 一、项目概述

**AI Companion** 是一款运行在 Android 手机上的 AI 陪伴聊天应用。用户与预设的 AI 角色（"小美"）进行对话，AI 会记住用户说过的重要信息，下次聊天时能主动回忆。

### 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| Android UI | Kotlin + 原生 View (XML) | 聊天界面、设置面板 |
| Python 运行时 | Chaquopy 15.0.1 | 在 APK 内嵌入 Python 3.10 |
| AI 接口 | DeepSeek API (OpenAI 兼容) | Chat (对话) + Embedding (向量) |
| 记忆存储 | SQLite (WAL 模式) | 持久化向量 + 元数据 |
| 构建 | Gradle 8.x + AGP 8.x | APK 构建 |

### 核心设计：为什么用 Python？

角色扮演引擎和记忆系统逻辑复杂（Prompt 构建、语义检索、时间衰减、记忆提取），用 Python 更灵活。通过 Chaquopy，Python 代码直接打包进 APK，无需网络下载运行时。

#### 3.2.4 对话流程（role_player.py）
```

        → deepseek.chat() 调用 API
        → 解析回复 → context_manager.record() 存入历史

```

对话流程中已嵌入记忆系统的两个钩子：
- **对话前**：`inject_memories(text)` → 检索相关记忆 → 注入 Prompt
- **对话后**：`_auto_remember()` → 提取新记忆 → 存储到 SQLite

### 3.3 记忆系统

#### 3.3.1 总体架构

```
MemoryOrchestrator（编排层）
    ├── Extractor     ← 从对话中提取记忆条目
    ├── Embedder      ← 调用 DeepSeek Embedding API
    ├── VectorStore   ← SQLite 持久化存储
    ├── Retriever     ← 混合检索
    ├── Decay         ← 时间衰减计算
    └── Exporter      ← JSON 导出
```

#### 3.3.2 记忆条目结构
```python
{
    "id": "abc123",          # UUID
    "type": "fact",          # fact（事实）/ event（事件）
    "content": "用户叫小明",
    "embedding": [0.1, ...], # 1536 维向量
    "importance": 0.7,       # 0-1
    "created_at": 1717920000,
    "last_accessed_at": 1717923600,
    "access_count": 5,
    "decay_factor": 1.0,     # 0-1，越低越不重要
    "source_turn": 10        # 来源对话轮次
}
```

#### 3.3.3 存储层（vector_store.py）
- **SQLite 持久化**：WAL 模式，线程安全，写入不阻塞读取
- **倒排索引**：token → 文档 ID 映射，用于关键词预过滤
- **全量余弦搜索**：当倒排结果不足时回退到全量向量搜索
- **并发安全**：`threading.Lock` 保护写操作
- **关闭防护**：`close()` 后所有操作返回空列表（防 crash）

#### 3.3.4 检索层（retriever.py）
混合检索策略（3 步流水线）：

```
1. Embedder.embed(query) → 查询向量
2. VectorStore.search(vector, top_k*2) → 倒排预过滤 → 候选集
3. 余弦相似度精排 + 时间衰减加权 → 最终 top_k
```

不复用 VectorStore 返回的相似度分数，避免双重计算（已修复）。

#### 3.3.5 提取层（extractor.py）
两种模式：

**规则模式**（MVP 默认，零 API 调用）：
- 正则匹配中文事实和事件
- 支持类型：姓名、喜好、居住地、工作、目标、计划、事件等
- 例："我喜欢打篮球" → `{type: "fact", content: "用户喜欢打篮球"}`

**LLM 模式**（已实现，待启用）：
- 调用 DeepSeek API 智能提取
- 适合复杂/隐含信息
- 每 N 轮对话节流控制（MVP 中 `N = 2**63` 即禁用）

**去重**（两种方式）：
1. bigram Jaccard 相似度快速过滤
2. Embedding 余弦相似度精确去重

#### 3.3.6 衰减算法（decay.py）
- **指数衰减**：`e^(-λt)`，半衰期按类型：事实 30 天 / 事件 7 天 / 短期 180 天
- **访问 boost**：每次检索命中，衰减因子 +5%
- **get_weight()**：优先使用预计算的 `entry.decay_factor`，避免重复计算

#### 3.3.7 编排层（orchestrator.py）🆕
核心方法：

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `remember()` | user_text, ai_text, turn_id | (extracted, stored, skipped) | 提取+去重+存储 |
| `recall()` | query_text, n | MemoryEntry 列表 | 检索+衰减更新 |
| `inject_prompt()` | query_text | Prompt 片段 | 格式化为 Prompt 文本 |
| `get_stats()` | — | {total, by_type, ...} | 统计分析 |

#### 3.3.8 Kt ↔ Py 通信桥（chat_bridge.py）
对外暴露的 **8 个接口**：

| 接口 | 方向 | 参数 | 返回值 |
|------|------|------|--------|
| `set_api_key(key)` | Kt→Py | API Key 字符串 | `{"status": "ok"}` |
| `init()` | Kt→Py | preset, model | `{"status":"ok","card":{...}}` |
| `chat(text)` | Kt→Py | 用户消息 | `{"status":"ok","reply":"..."}` |
| `init_memory(db_path)` | Kt→Py | SQLite 路径 | `{"status":"ok","memory_count":N}` |
| `inject_memories(text)` | Kt→Py | 当前消息 | `{"status":"ok","count":N}` |
| `get_memory_stats()` | Kt→Py | — | `{"status":"ok","total":N,...}` |
| `inject_memories(text)` | Kt→Py | query | 注入结果 dict |
| `get_memory_stats()` | Kt→Py | — | 统计数据 dict |

## 八、关键技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| Android UI 方案 | 传统 View (XML) | 简单、兼容性好，项目规模不大 |
| Python 嵌入方案 | Chaquopy | 唯一成熟方案，可嵌入纯 Python 依赖 |
| 向量存储 | SQLite + 手动余弦 | 无需 ChromaDB/Pinecone，离线可用 |
| 记忆提取 | 规则 + LLM 双模式 | 规则免费快速，LLM 智能准确 |
| 混合检索 | 倒排 → 余弦精排 | 平衡准确率和性能 |
| ABI 过滤 | 仅 arm64-v8a | 减小 APK 体积，避免 Chaquopy bug |
| 日志系统 | loguru → 标准logging降级 | Android 环境兼容 |

| 记忆全量搜索 OOM | 记忆量 > 2000 可能卡顿 | P1 待修复：分页加载 |
| Embed API 无缓存 | 每次检索增加 200-500ms | P1 待修复：LRU 缓存 |
| 无离线功能 | 断网无法对话 | 设计上依赖云端 AI |
| MIUI 后台限制 | APP 可能被系统杀掉 | 用户可关闭省电限制 |
