## 修复清单（7 项全部完成）

### 1. vector_store.py — close() 防护
- **新增** `self._closed` 标志（`__init__` 初始化为 `False`）
- **新增** `_check_closed()` 方法，所有 10 个公开方法入口检查
- **修改** `close()` 设置 `_closed = True`，关闭前执行 checkpoint

### 2. vector_store.py — delete() TOCTOU 竞态
- **修改前**：先 `get()`（有锁→释放） → 再 `delete()`（重新获取锁），中间有时间窗口
- **修改后**：get + delete + 索引更新合并入同一个 `with self._lock` 块

### 3. vector_store.py — 错误传播
- `_get_by_ids()`：数据库错误时 `raise MemoryStorageError`（之前静默返回 `[]`）
- `_get_all()`：同上
- `count()`：同上（之前返回 `0`）
- `get_by_type()`：同上（之前返回 `[]`）

### 4. vector_store.py — WAL checkpoint
- **新增** `_checkpoint()` 方法：执行 `PRAGMA wal_checkpoint(TRUNCATE)`
- **新增** `_add_count` 计数器，每 100 次 `add()` 触发一次 checkpoint
- `close()` 时也执行 checkpoint

### 5. vector_store.py — search() 返回带分数结果
- **修改前**：返回 `list[MemoryEntry]`（丢弃了余弦相似度分数）
- **修改后**：返回 `list[tuple[MemoryEntry, float]]`，每项自带 (entry, similarity_score)
- 默认 top_k 从 5 调整为 10（提供更多候选给上层过滤）

### 6. decay.py — get_weight 优先使用 entry.decay_factor
- **修改前**：每次都调用 `calculate_decay()` 重新计算
- **修改后**：当 `entry.decay_factor > 0 and < 1.0`（已通过 `update_decay()` 设置过），直接使用预计算值
- 仅当 decay_factor 为初始值 1.0 时才重新计算

### 7. retriever.py — 消除余弦相似度双重计算
- **修改前**：`VectorStore.search()` 算一遍 → `Retriever.retrieve()` 再算一遍
- **修改后**：直接使用 `search()` 返回的分数列表，仅做衰减加权
- 清理了未使用的 `cosine_similarity` 导入（`retrieve_by_type` 仍需保留）

### 8. extractor.py — 去重优先语义向量
- **新增**：当新旧记忆都有 embedding 时，优先用余弦相似度去重
- **回退**：无 embedding 时仍用 bigram Jaccard（规则提取场景）
- **优化**：已有记忆按类型分组，减少 O(n*m) 中的 n

### 9. exporter.py — 移除未实现的 batch_size 参数
- 移除 `export_chat_history` 和 `export_memories` 的 `batch_size` 参数
- 移除 `_DEFAULT_BATCH_SIZE` 常量
