## P5 主动推送

### T-P5-01: chat_bridge 添加 generate_proactive_message() 接口

| 属性 | 内容 |
|------|------|
| **优先级** | 高 |
| **负责** | @backend-dev |
| **依赖** | 无（engine.py 已就绪） |
| **文件** | chat_bridge.py |

**需求**：在 `chat_bridge.py` 中新增 `generate_proactive_message()` 函数，供 WorkManager/ProactiveWorker 通过 Chaquopy 调用。

**实现要点**：
1. 从 `_ctx` 获取 player 和 orchestrator
2. 创建 `ProactiveEngine` 实例
3. 调用 `engine.decide_and_generate()` 决策并生成消息
4. 如果返回消息，同时存入对话上下文（`_auto_remember`）
5. 返回 JSON 格式结果

**完成标准**：
- Kotlin 端可调用此接口
- 返回格式：`{"status": "ok", "title": "...", "message": "..."}` 或 `{"status": "skip", "message": "跳过原因"}`
- 消息风格符合角色卡"小美"

### T-P5-02: MainActivity 配置 WorkManager 定时任务

| 属性 | 内容 |
|------|------|
| **优先级** | 高 |
| **负责** | @frontend-dev |
| **依赖** | T-P5-01 |
| **文件** | MainActivity.kt |

**需求**：在 `MainActivity.kt` 中配置 WorkManager 定期调度 ProactiveWorker。

**实现要点**：
1. 导入 `androidx.work.*` 相关类
2. 在 `onCreate()` 或 `initPython()` 成功后调用 `scheduleProactiveWorker()`
3. 使用 `PeriodicWorkRequestBuilder` 设置周期（默认 3 小时）
4. 添加网络约束（`setConstraints`）

**完成标准**：
- WorkManager 正常调度，logcat 可见 "ProactiveWorker doWork 开始"
- 收到通知后点击可进入 APP

## P6 记忆增强

### T-P6-01: 完善 Android 端记忆管理页面

| 属性 | 内容 |
|------|------|
| **优先级** | 高 |
| **负责** | @frontend-dev |
| **依赖** | 无（chat_bridge 接口已就绪） |
| **文件** | MainActivity.kt、新建议管理 Activity/Dialog |

**需求**：将当前简陋的"记忆统计"对话框升级为完整的记忆管理界面。

**实现要点**：
1. 记忆列表展示（RecyclerView，显示类型标签+内容预览+时间）
2. 支持分页加载（`list_memories(page, page_size)`）
3. 单条删除（长按弹出确认，调用 `delete_memory(memory_id)`）
4. 清空全部（按钮 + 二次确认，调用 `clear_memories()`）
5. 关键字搜索（搜索框，调用 `search_memories(keyword)`）

**完成标准**：
- 可以看到所有记忆的列表
- 可以搜索、删除、清空记忆
- 空状态提示友好
- UI 不卡顿
