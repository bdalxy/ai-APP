## 三、P5 阶段：主动推送/通知

### 3.1 功能描述

让 AI 角色能够主动发起消息，模拟真人社交互动。用户不需要每次都由自己开启对话，AI 会在合适的时机"找你聊天"。

**核心场景**：
- 早上起来，小美发来问候："早安呀~今天天气不错呢！"
- 用户长时间没说话，小美轻轻提醒："在忙吗？想你了~"
- 特定时间/事件触发（如生日祝福）

### 3.2 技术方案

#### 方案选型

考虑到项目约束（自用 APP，不依赖云服务），选择 **本地轮询 + Android AlarmManager** 方案：

```
AlarmManager 定时唤醒
    ↓
BroadcastReceiver / WorkManager
    ↓
Python 端 ProactiveEngine.decide()
    ↓ (决定是否发送)
Python 端 ProactiveEngine.generate()
    ↓ (调用 DeepSeek 生成消息)
Android NotificationManager
    ↓
系统通知栏推送
    ↓ (用户点击)
打开 APP → 进入聊天
```

**为什么不选 FCM**：
- FCM 在国内不稳定
- 需要额外服务器
- 自用 APP 用 AlarmManager 足够

**为什么用 WorkManager 而不是 AlarmManager 直接**：
- WorkManager 兼容 Doze 模式
- 支持约束条件（网络可用时才执行）
- API 更现代

| **依赖** | P4 手机验证全部通过 |
| **估时** | 2 小时 |
| **文件** | `orchestrator.py`、`extractor.py`、`chat_bridge.py` |

**工作内容**：
1. 启用 `_extract_interval = 5`（每 5 轮启用一次 LLM 提取）
2. LLM 提取与规则提取并行：规则提取每次都执行，LLM 提取每隔 N 轮补充
3. 在设置面板中增加 "记忆提取模式" 选项：
   - 仅规则（默认，免费）
   - 混合（每 5 轮 LLM）
   - 智能（仅 LLM，费用较高）

**完成标准**：
- 规则模式：与当前行为一致
- 混合模式：第 1/6/11... 轮使用 LLM 提取
- LLM 提取的质量明显优于规则（能提取隐含信息）
- 成本可追踪
