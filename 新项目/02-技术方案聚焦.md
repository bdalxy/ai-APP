## 二、API 调用成本分析

### DeepSeek 当前价格（近似）

| 类型 | 价格 |
|------|------|
| 输入 | ~¥1 / 100万 token |
| 输出 | ~¥2 / 100万 token |

**换算**：1 条消息约 50-200 token，1 次对话（20 轮）约 2000-4000 token。

### 各项功能成本估算

| 场景 | 每次消耗 | 每次成本 | 假设频率 | 月成本 |
|------|----------|----------|----------|--------|
| 普通聊天（20轮） | 输入 3000 + 输出 500 token | ¥0.004 | 10 次/天 | ¥1.20 |
| 记忆提取（LLM） | 输入 1000 + 输出 300 token | ¥0.0016 | 混合后 2 次/天 | ¥0.10 |
| 主动消息生成 | 输入 500 + 输出 150 token | ¥0.0008 | 3 次/天 | ¥0.07 |
| 联网搜索增强 | 输入 2000 + 输出 500 token | ¥0.003 | 1 次/天 | ¥0.09 |

**预估月成本：约 ¥1.50 - ¥5.00**（取决于使用频率）

### 混合提取 vs 纯 LLM 提取

| | 纯 LLM 提取 | 混合提取 |
|------|-------------|----------|
| 触发时机 | 每次对话结束 | 仅重要对话 + 用户标记 |
| 日提取次数 | ~10 次 | ~2 次 |
| 月成本 | ~¥0.50 | ~¥0.10 |
| 准确率 | 高 | 中高（重要信息用 LLM 保证准确） |
| 遗漏风险 | 低 | 低（规则能兜底常见模式） |

**结论：混合方案足够，成本几乎可忽略。**

## 三、主动消息引擎设计（最终版）

### 用户配置项

```json
{
  "proactive": {
    "wake_time": "08:00",        // 起床时间
    "sleep_time": "23:00",       // 睡觉时间
    "interval_minutes": 120,     // 消息间隔（分钟）
    "random_factor": 0.3         // 随机偏移（±30%）
  }
}
```

### 工作流程

```
┌─────────────────────────────────────────────────┐
│                  一天的时间线                     │
│                                                 │
│  00:00 ─────── wake_time ─────── sleep_time ───│
│    │              │                  │           │
│    │  休息时段     │   活跃时段        │  休息时段  │
│    │  默认不发     │   按间隔随机发    │  默认不发  │
│    │  除非角色卡   │                  │  除非角色卡 │
│    │  强烈想发     │                  │  强烈想发  │
│    │  (受间隔限制) │                  │  (受间隔限制)│
│    │              │                  │           │
│    └──────────────┴──────────────────┘           │
│                                                 │
│  活跃时段示例 (interval=120分钟):                 │
│  08:00 起床消息 → 10:15 随机 → 12:40 随机       │
│  → 14:50 随机 → 17:05 随机 → 19:30 随机         │
│  → 21:45 随机 → 23:00 晚安消息                   │
└─────────────────────────────────────────────────┘
```

### 决策逻辑

```python
def should_send_message(now, config, character_card, last_message_time):
    # 1. 间隔检查
    if (now - last_message_time) < config.interval_minutes:
        return False

    # 2. 时间窗口检查
    if is_rest_period(now, config.wake_time, config.sleep_time):
        # 休息时段：只有角色卡强烈想发才发
        if character_card.proactive_urgency < 0.8:
            return False

    # 3. 随机因子
    base_interval = config.interval_minutes
    random_offset = base_interval * config.random_factor * random.uniform(-1, 1)
    actual_interval = base_interval + random_offset

    if (now - last_message_time) < actual_interval:
        return False

    # 4. 起床/睡觉特殊消息
    if is_wake_time(now, config.wake_time) and not wake_message_sent_today:
        return True, "wake_greeting"
    if is_sleep_time(now, config.sleep_time) and not sleep_message_sent_today:
        return True, "sleep_greeting"

    return True
```

## 四、记忆系统设计（最终版）

### 三层记忆 + 强化记忆

```
┌─────────────────────────────────────────┐
│           短期记忆（Short-term）          │
│  最近 50 轮对话，直接拼入上下文            │
│  每轮对话后自动更新                       │
└────────────────┬────────────────────────┘
                 │ 定期筛选、压缩
┌────────────────▼────────────────────────┐
│           长期记忆（Long-term）            │
│  ┌──────────────────────────────────┐   │
│  │ 普通记忆                           │   │
│  │ 规则提取（免费）+ LLM 提取（重要对话）│   │
│  │ 时间衰减：分数 = 重要性 × e^(-λt)   │   │
│  ├──────────────────────────────────┤   │
│  │ 强化记忆（Pinned）⭐                │   │
│  │ 用户明确说"记住这个"                │   │
│  │ AI 角色卡认定重要                   │   │
│  │ 不衰减 / 衰减极慢（λ=0.01）         │   │
│  │ 检索时权重 × 2.0                  │   │
│  └──────────────────────────────────┘   │
└────────────────┬────────────────────────┘
                 │ 关联分析
┌────────────────▼────────────────────────┐
│         关联记忆（Associative）           │
│  记忆之间的关联关系                       │
│  例如："喜欢猫" ← 关联 → "去过猫咖"       │
│  初期简化：基于关键词/向量相似度自动关联    │
└─────────────────────────────────────────┘
```

### 强化记忆的触发方式

1. **用户主动标记**：输入框旁边加一个 📌 按钮，点击后标记当前对话为"重要"
2. **AI 自动判断**：角色卡根据自身设定，对某些信息自动加权（如"女友型"角色对用户健康信息自动加权重）
3. **关键词触发**：检测到"记住"、"别忘了"、"很重要"等关键词时自动标记
