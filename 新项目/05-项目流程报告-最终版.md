## 1. 项目概述

### 1.1 项目定位

一个**纯本地运行的 Android AI 角色扮演聊天 APP**，单 APK 安装即用。核心目标是让 AI 角色像真人一样与用户聊天，具备记忆、主动发起话题等能力。

### 1.2 核心约束

| 约束 | 说明 |
|------|------|
| 无服务器 | 不能依赖任何服务器，纯本地运行 |
| 单 APK | 一个安装包，安装即用 |
| 低成本 | 典型月成本 ¥1-3，上限 ¥5 |
| 好架构 | 可维护、可扩展 |
| 初始 APK | 目标 < 80MB（含 Python 运行时），后续按需扩展 |
| 代码分工 | AI（Trae）全权编写，用户负责需求定义和测试验收 |

### 1.3 核心结论

> **四方审核一致通过：项目方案成熟、风险可控，具备进入开发阶段的条件。**

## 2. 需求确认

### 2.1 三大核心体验

#### 聊天质量 — "像真人在微信上聊天"
- 多段短对话为主，偶有长对话
- 角色卡驱动，人设稳定不崩塌
- 对话风格自然，避免机械感

#### 记忆系统 — "像真人的记忆"

```
短期记忆：最近 N 轮对话（可配置，默认 20 条）
    ↓ 定期筛选
长期记忆：
  ├─ 普通记忆：规则提取 + LLM 提取，时间衰减
  └─ 强化记忆：用户/AI 标记，几乎不衰减，检索权重 ×2
    ↓ 关联分析
关联记忆：记忆之间的关联关系（初期简化为同 session 关联）
```

**⚠️ 关键约束**：用户有大量聊天需求，记忆量可达 K 级（数千条），需要倒排索引 + 关键词预过滤 + 定期归档。

#### 主动消息 — "AI 能主动找你聊天"
- 用户设定起床/睡觉时间，划定休息时段和活跃时段
- 活跃时段按间隔随机发送
- 休息时段默认不发（除非角色卡强烈想发）
- 起床/睡觉时有特殊问候消息

### 2.2 角色卡 + 世界书

**角色卡**：JSON 格式，定义角色性格、背景、对话风格、记忆关注点、主动消息风格。

**世界书**：完全参考 Tavo APP 的格式，支持正则表达式触发。默认"三次元现实世界"，用户可自行添加。

### 2.3 设置系统

用户可自行配置的完整设置项（16 项）：

首次启动时弹出引导流程（3 步）：

### 3.3 向量检索方案（针对 K 级记忆）

**核心方案**：DeepSeek Embedding API + 倒排索引预过滤 + 纯 Python 余弦相似度

```
用户消息 → 关键词提取(bigram) → 倒排索引预过滤(50ms) → Top-500候选集 → 余弦相似度精排(50ms) → Top-K结果
```

**性能预估**：

| 数据量 | 全量扫描耗时 | 索引预过滤后耗时 |
|--------|-------------|-----------------|
| 5,000 | ~500ms | ~50ms |
| 10,000 | ~1,000ms | ~80ms |
| 20,000 | ~2,000ms | ~120ms |

**定期归档**：超过 30 天未使用的记忆自动归档到 `data/archives/archive_*.json`。

### 5.2 记忆系统（K 级优化）

#### 倒排索引 + 向量检索

```python
class MemoryRetriever:
    """混合检索器：倒排索引预过滤 + 余弦相似度精排"""
    
    def retrieve(self, query_vector, query_text, top_k=5):
        # 1. 关键词提取（bigram）
        keywords = self._extract_keywords(query_text)
        
        # 2. 倒排索引预过滤（候选集 50-500 条）
        candidates = self.inverted_index.search(keywords)
        
        # 3. 余弦相似度精排
        scored = self._cosine_rank(query_vector, candidates)
        
        # 4. 时间衰减 + 强化记忆加权
        scored = self._apply_decay_and_boost(scored)
        
        return scored[:top_k]
```

#### 正则匹配引擎

```python
class WorldBookEngine:
    def match_and_inject(self, text, round_num):
        matched = []
        for entry in self.entries:
            # 1. constant → 始终注入
            if entry.constant:
                matched.append(entry)
                continue
            
            # 2. 正则匹配
            if entry.regex_keys:
                if any(re.search(pattern, text) for pattern in entry.regex_patterns):
                    matched.append(entry)
                    continue
            
            # 3. 关键词匹配
            if self._match_keywords(entry, text):
                # 4. 概率检查
                if random.randint(1, 100) <= entry.probability:
                    # 5. 冷却检查
                    if self._check_cooldown(entry, round_num):
                        matched.append(entry)
        
        # 按优先级排序，返回注入内容
        matched.sort(key=lambda e: e.priority, reverse=True)
        return [e.content for e in matched[:10]]
```

#### 正则安全（ReDoS 防护）

```python
import re
import time

REDOS_PATTERNS = [
    r'\(.+\+\)\+',      # 嵌套量词
    r'\(.\*\).\*',      # 嵌套星号
    r'\(.\+.\+\)',      # 嵌套加号
]

def is_safe_regex(pattern: str, timeout: float = 0.1) -> bool:
    """检测正则表达式是否安全（无 ReDoS 风险）"""
    # 1. 静态检测：匹配已知危险模式
    for danger in REDOS_PATTERNS:
        if re.search(danger, pattern):
            return False
    
    # 2. 动态检测：超时则拒绝
    try:
        compiled = re.compile(pattern)
        # 用短字符串测试，设置超时
        start = time.time()
        compiled.search("test_string_" * 10)
        if time.time() - start > timeout:
            return False
    except re.error:
        return False
    
    return True
```

### 7.2 离线兜底方案

当网络不可用或 API 失败时：
- 返回预设回复：`"我现在有点断线，稍后再聊好吗？"`
- 记录重试队列，网络恢复后自动重试
- 离线时主动消息功能暂停

### 7.3 关键边界条件

- **空状态**：首次安装、无记忆、无对话历史
- **极限状态**：对话 500+ 轮、记忆 20,000 条、单条消息 5000 字
- **异常输入**：纯表情、Unicode 特殊字符、注入攻击、ReDoS 正则
- **状态切换**：WiFi/4G 切换、息屏/亮屏、APP 后台/前台、手机重启
- **并发**：用户快速发送 + 主动消息同时到达
