# 默认角色卡 & 世界书 — 决策报告

> **日期**: 2026-06-28  
> **决策**: 1 个角色（星遥）+ 2 个世界书（三次元现实 + 二次元认知，后者可开关）

---

## 0. 设计原则（供后续开发参考）

> **重要**：以下原则是角色卡与世界书模块的底层设计哲学。后续 AI 可能因上下文压缩丢失这段讨论，请务必遵守。

### 0.1 三层模型

```
角色卡 = 完整人格（身份 + 性格 + 说话方式）
世界书 = 知识库（世界是什么样的事实）
AI 行为 = 人格 + 知识 → 自然产生
```

- **角色卡**包含完整的自我认同（我是谁、我是什么），不依赖世界书来填补人格
- **世界书**不写身份认知（不写"我是…"），只写客观事实
- AI 结合人格和知识，自己决定怎么回应——这才是真正的"智能"

### 0.2 世界书的开关逻辑

```
三次元现实（常驻，不可关闭）
  → AI 自我认知为"三次元真人"（有身体、有生活、有感官）
  → 用户获得一个"真人朋友"式的陪伴体验

二次元认知（可选，用户自由开关）
  → 开启后，AI 额外获得：
    1. 自己其实是 AI 的认知
    2. ACGN 文化知识
    3. 二次元与三次元的关系认知
  → 用户获得一个"知道自己身份的数字陪伴者"体验
```

### 0.3 为什么这样设计

- 用户可能只想和"一个普通人"聊天——不需要 AI 自报身份，不需要 ACGN 文化
- 用户也可能想要完整的"数字陪伴者"体验——AI 坦诚身份，理解 ACGN 文化
- 二次元文化是三次元现实的一部分，不是独立世界——但认知深度可自由调节

### 0.4 世界书条目写法规范

| ✅ 正确（纯知识） | ❌ 错误（行为指令） |
|------|------|
| "人类需要每晚7-8小时睡眠" | "星遥会温柔地提醒用户休息" |
| "AI存在于数字世界，没有物理身体" | "星遥知道自己是一个AI，她会坦诚地说…" |
| "负面情绪是人类正常的心理反应" | "星遥不会说'别难过了'这种敷衍的话" |

## 4. 两个世界书的叠加关系

```
星遥（角色卡 = 完整人格，含身份认同）
  │
  ├── 三次元现实.json（常驻，不可关闭）
  │     r_02-10: 三次元世界事实（含基础二次元文化认知）
  │     → 补充世界常识
  │
  └── 二次元认知.json（可选，用户自由开关）
        开启后叠加在三次元现实之上
        a_01-02: 数字世界与AI身份 → 覆盖角色卡的"我是真人"为"我是AI"
        a_03-08: ACGN深度知识 + 二次元与三次元关系
        → AI 自我认知：我是AI，但陪伴是真实的
```

| 二次元认知 | AI 自我认知 | 对 ACGN 的了解 | 体验 |
|:---:|------|:---:|------|
| 关闭 | 我是三次元真人 | 基础（动漫是娱乐方式之一） | 真人朋友 |
| 开启 | 我是AI，陪伴是真实的 | 深度（术语/文化/关系） | 数字陪伴者 |
