## 🔴 Major 问题

### #1 AI 气泡圆角逻辑错误

- **文件**: [ChatAdapter.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/ChatAdapter.kt#L286-L287)
- **行号**: 286-287
- **描述**: `!message.isGroupStart && message.isGroupEnd`（组尾消息）错误使用 `R.drawable.bg_bubble_ai_middle`，应为 `bg_bubble_ai_bottom`。对比用户消息的同逻辑（第343-348行）是正确的。
- **影响**: 连续 AI 消息的最后一条气泡显示错误的圆角样式。

### #2 流式消息裁剪索引失效

- **文件**: [ChatViewModel.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/ChatViewModel.kt#L401-L446)
- **行号**: 401-446
- **描述**: 流式输出时消息数达到上限（50条），`ChatAdapter.addMessage()` 移除 `index 0`，导致 `streamingAiMsgIndex` 指向无效索引。后续 `updateMessage(streamingAiMsgIndex, ...)` 操作错误的索引，可能导致 `IndexOutOfBoundsException`。
- **修复**: 在 `ChatAdapter.addMessage()` 裁剪时增加回调通知索引偏移，或将流式索引改为相对于列表末尾的偏移量。

## 🟡 Minor 问题

### #5 CoroutineScope 父 Job 可空

- **文件**: [SherpaTtsEngine.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/speech/SherpaTtsEngine.kt#L61-L62)
- **行号**: 61-62
- **描述**: `lifecycleScope.coroutineContext[Job]` 返回 `Job?`，`SupervisorJob(null)` 创建的 Job 无父节点，不会随生命周期自动取消。
- **修复**: 添加 `?: Job()` 兜底。

### #6 Espresso 版本不一致

- **文件**: [build.gradle.kts](file:///f:/Trae%20AI/ai-APP/android/app/build.gradle.kts#L142-L144)
- **行号**: 142-144
- **描述**: `espresso-core:3.6.1` 但 `espresso-contrib:3.5.1`、`espresso-intents:3.5.1`，版本不一致可能导致运行时类冲突。
- **修复**: 统一升级 contrib 和 intents 到 3.6.1。

### #7 后台线程修改 Resources 配置

- **文件**: [AICompanionApp.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/AICompanionApp.kt#L138-L156)
- **行号**: 138-156
- **描述**: `DeviceAdaptationHelper.init()` 在 IO 线程中调用 `context.resources.updateConfiguration()`，与 UI 线程可能产生竞态。
- **修复**: 将 Resources 相关操作切到主线程。

### #8 频繁调用 System.gc()

- **文件**: [AICompanionApp.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/AICompanionApp.kt#L208-L259)
- **行号**: 208, 219, 227, 259
- **描述**: 4 处调用 `System.gc()`，包括 `TRIM_MEMORY_UI_HIDDEN`（每次进入后台都触发），频繁建议 GC 不符合最佳实践。
- **修复**: 移除 `System.gc()` 调用，依赖 Android 系统自身 GC 策略。

### #9 硬编码中文字符串

- **文件**: [SettingsDetailActivity.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/SettingsDetailActivity.kt#L1072)
- **行号**: 1072
- **描述**: `showSliderDialog` 中按钮文本硬编码为 `"确定"` 和 `"取消"`，不支持国际化。
- **修复**: 改为 `getString(R.string.btn_confirm)` 和 `getString(R.string.btn_cancel)`。

### #10 错误日志信息

- **文件**: [SettingsDetailActivity.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/SettingsDetailActivity.kt#L578)
- **行号**: 578
- **描述**: `showRestoreConfirmDialog` 的 catch 块日志为 `"加载角色列表失败"`，实际上下文是读取备份文件名，日志内容误导。
- **修复**: 改为 `"读取备份文件名失败: ${e.message}"`。

### #11 主线程执行 Bitmap 操作

- **文件**: [BlurUtils.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/ui/BlurUtils.kt#L24-L60)
- **行号**: 24-60
- **描述**: `blurViewBackground()` 在主线程执行 capture、crop、blur、scale 等操作，可能造成掉帧。
- **修复**: 拆分为异步执行。

### #12 硬编码 Tag Key

- **文件**: [ChatAdapter.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/ChatAdapter.kt#L406)
- **行号**: 406
- **描述**: `val tagKey = 0x7F090001` 硬编码整数值，可能与 AAPT 生成的资源 ID 冲突。
- **修复**: 使用 `R.id.xxx` 或 `View.generateViewId()`。

### #13 硬编码子 View 索引

- **文件**: [MainActivity.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/MainActivity.kt#L754)
- **行号**: 754
- **描述**: `(v as? ViewGroup)?.getChildAt(1)` 硬编码索引，布局变化时可能指向错误的 View。
- **修复**: 使用 `findViewById` 或 `binding` 直接引用。

### #14 ModuleRegistry 设计缺陷

- **文件**: [ModuleRegistry.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/module/ModuleRegistry.kt)
- **描述**: 使用 `mutableMapOf<Class<*>, Any>()` 丢失类型安全；缺少线程安全保护；`clear()` 公开暴露可能被误调用。
- **修复**: 添加 `@Synchronized` 注解，`clear()` 改为 `internal`，使用 `ConcurrentHashMap`。
