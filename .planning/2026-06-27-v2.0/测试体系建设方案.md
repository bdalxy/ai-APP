## 2. 测试框架选型

### 2.1 JUnit 5 vs JUnit 4

| 对比维度 | JUnit 4 | JUnit 5 |
|----------|---------|---------|
| 参数化测试 | `@RunWith(Parameterized.class)` | `@ParameterizedTest` + `@ValueSource` 等 |
| 测试分组 | 无原生支持 | `@Tag("unit")` / `@Tag("integration")` |
| 扩展模型 | `@RunWith` + `@Rule` | `@ExtendWith` + Extension API |
| 生命周期 | `@Before` / `@After` | `@BeforeEach` / `@AfterEach` + `@BeforeAll` / `@AfterAll` |
| 嵌套测试 | 无 | `@Nested` 内嵌类 |
| 断言 | `Assert.assertEquals()` | `Assertions.assertEquals()` + 丰富断言 API |
| AGP 兼容性 | 默认支持 | 需额外插件 |

**决策：采用 JUnit 5**

理由：
- 更好的参数化测试支持（适合记忆系统、世界书等需要多组输入的模块）
- `@Nested` 测试组织（将相关测试分组，可读性更好）
- `@Tag` 分组（单元测试 / 集成测试 / 慢速测试分类执行）
- JUnit 5 是 Android 官方推荐方向（AndroidX Test 2.x 已全面支持）

### 2.2 MockK vs Mockito

| 对比维度 | Mockito | MockK |
|----------|---------|-------|
| Kotlin 原生支持 | 需 `mockito-kotlin` 辅助 | 原生 Kotlin DSL |
| `final` 类 mock | 需 `mockito-inline` | 原生支持 |
| 协程 suspend 函数 mock | 需额外处理 | `coEvery {}` / `coVerify {}` 原生支持 |
| object / companion object mock | 不支持 | `mockkObject()` 支持 |
| 语法风格 | Java 风格 | Kotlin 风格 |
| 社区活跃度 | 高 | 中高 |

**决策：采用 MockK**

理由：
- 项目是 Kotlin 项目，MockK 提供 Kotlin 原生 DSL
- `coEvery` / `coVerify` 对协程的一等支持（ChatViewModel 大量使用协程）
- 可 mock Kotlin `object` 和 `companion object`
- 语法更简洁，如 `every { obj.method() } returns value`

### 2.4 是否需要 Robolectric

| 对比维度 | Robolectric | Android Instrumentation Test |
|----------|-------------|------------------------------|
| 运行速度 | 快（秒级，运行在 JVM） | 慢（分钟级，需要 APK 安装到设备） |
| 需要设备/模拟器 | 不需要 | 需要 |
| CI 支持 | 简单（标准 JVM 环境） | 需要 Android SDK + 模拟器 |
| 真实度 | 模拟 Android 环境 | 真实 Android 环境 |
| Python 桥接 (Chaquopy) | 不支持 | 支持 |
| Sherpa-ONNX 本地库 | 不支持 | 支持 |

**决策：采用 Robolectric + 仪器化测试 分层策略**

- **Robolectric**：纯 Kotlin 单元测试（ViewModel、工具类、纯逻辑）
- **仪器化测试**：涉及 Python 桥接、TTS、ONNX 的测试（需要真实设备）

### 2.5 Python 测试策略

Chaquopy 的 Python 代码运行在 Android 设备上，无法直接在本地 pytest 中运行。采用以下策略：

| 方案 | 说明 |
|------|------|
| **纯逻辑测试（推荐）** | 抽取不依赖 Android 环境的纯 Python 逻辑到独立模块，使用标准 `unittest` / `pytest` 在 PC 端测试 |
| **Chaquopy 仪器化测试** | 通过 Kotlin 仪器化测试调用 Python 模块，验证端到端行为 |
| **PC 端模拟测试** | 在 PC 开发机上直接运行 Python 代码（需安装依赖），使用 pytest 测试 |

**可测试的纯 Python 逻辑（无需 Android 环境）：**

- `src/utils/text_utils.py` — 文本相似度计算
- `src/utils/lru_cache.py` — LRU 缓存
- `src/utils/retry.py` — 重试逻辑
- `src/memory/bm25.py` — BM25 评分（纯算法）
- `src/memory/decay.py` — 时间衰减计算（纯算法）
- `src/memory/memory_types.py` — 冲突检测、实体提取（纯算法）
- `src/chat_engine/card_parser.py` — 角色卡解析（JSON 解析）
- `src/chat_engine/prompt_builder.py` — Prompt 组装（纯字符串拼接）
- `src/world_book/world_book.py` — 世界书匹配引擎（正则匹配算法）
- `src/plugins/plugin_base.py` — 插件基类逻辑

### 4.3 步骤 3：编写第一个单元测试

选择 `src/utils/text_utils.py` 的 `text_similarity()` 函数作为第一个测试目标——纯函数、无外部依赖、逻辑清晰。

**Python 测试文件：** `android/app/src/main/python/tests/test_text_utils.py`

```python
"""text_utils 模块单元测试。"""

import sys
import os
import pytest

# 将 src 目录加入路径
sys.path.insert(0, os.path.join(os.path.dirname(__file__), ".."))

from src.utils.text_utils import text_similarity


class TestTextSimilarity:
    """text_similarity 函数测试。"""

    def test_identical_texts(self):
        """完全相同的文本应返回 1.0。"""
        assert text_similarity("你好世界", "你好世界") == 1.0

    def test_completely_different(self):
        """完全不同的文本应返回 0.0。"""
        assert text_similarity("abc", "xyz") == 0.0

    def test_empty_strings(self):
        """空字符串应返回 0.0。"""
        assert text_similarity("", "hello") == 0.0
        assert text_similarity("hello", "") == 0.0
        assert text_similarity("", "") == 0.0

    def test_partial_overlap(self):
        """部分重叠的文本应返回 0 到 1 之间的值。"""
        result = text_similarity("你好世界", "你好地球")
        assert 0.0 < result < 1.0

    def test_single_char(self):
        """单字符文本无法形成 bigram，应返回 0.0。"""
        assert text_similarity("a", "a") == 0.0

    def test_with_punctuation(self):
        """标点符号应被过滤。"""
        # "hello, world!" 和 "hello world" 过滤标点后应相同
        result = text_similarity("hello, world!", "hello world")
        assert result == 1.0

    @pytest.mark.parametrize("text1,text2,expected", [
        ("abc", "abc", 1.0),
        ("abc", "abd", 0.5),  # bigrams: ab,bc vs ab,bd → 交集1/并集2
        ("中文测试", "中文考试", None),  # 部分重叠
    ])
    def test_parametrized(self, text1, text2, expected):
        """参数化测试多个输入组合。"""
        result = text_similarity(text1, text2)
        if expected is not None:
            assert result == expected
        else:
            assert 0.0 < result < 1.0
```

class CharacterActivityTest {

    @get:Rule

    }

        onView(withId(R.id.fabAddCharacter))
            .check(matches(isDisplayed()))
    }

    @Test
    fun emptyStateShowsWhenNoCharacters() {
        // 验证空状态提示可见（如果没有角色）

            onView(withId(R.id.tvEmptyHint))
                .check(matches(isDisplayed()))
        } catch (_: Exception) {
            // 如果已有角色，列表可见，空状态不可见，这也是合理的
            onView(withId(R.id.rvCharacters))
                .check(matches(isDisplayed()))
        }
    }

    @Test
    fun backButtonReturnsToMain() {
        // 点击返回按钮，应回到主界面
        onView(withId(R.id.btnBack))
            .perform(click())
    }

package com.aicompanion.app.chat

import com.aicompanion.app.ConversationCoordinator
import com.aicompanion.app.ConversationSession
import com.aicompanion.app.ConversationSessionManager
import io.mockk.*
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.ExperimentalCoroutinesApi
import kotlinx.coroutines.test.*
import org.junit.jupiter.api.*
import org.junit.jupiter.api.Assertions.*

@OptIn(ExperimentalCoroutinesApi::class)
class ConversationCoordinatorTest {

    private val testDispatcher = StandardTestDispatcher()

    private lateinit var sessionManager: ConversationSessionManager
    private lateinit var coordinator: ConversationCoordinator

    @BeforeEach
    fun setUp() {
        Dispatchers.setMain(testDispatcher)
        sessionManager = mockk(relaxed = true)
        coordinator = ConversationCoordinator(sessionManager)
    }

    @AfterEach
    fun tearDown() {
        Dispatchers.resetMain()
        clearAllMocks()
    }

    @Test
    fun `创建新会话 - 应调用 sessionManager`() = runTest {
        // Given
        val mockSession = mockk<ConversationSession>(relaxed = true)
        every { sessionManager.createSession(any()) } returns mockSession

        // When
        coordinator.startNewSession("character_1")

        // Then
        verify { sessionManager.createSession(match { it == "character_1" }) }
    }

    @Test
    fun `暂停会话 - 协程函数验证`() = runTest {
        // Given: mock 协程函数
        coEvery { sessionManager.saveSession(any()) } just Runs

        // When
        coordinator.pauseSession()

        // Then

def calculate_decay_score(timestamp: float, half_life: float, current_time: float) -> float:
