## 2. 学习

> 用"是什么→为什么→怎么选"结构，供用户学习理解

### 2.1 API Key 安全存储

**是什么**：API Key 是用户访问 DeepSeek AI 的"钥匙"，如果泄露，别人可以用你的 Key 调用 AI，产生费用。

**为什么重要**：本项目是"API 用户自设"模式，Key 是用户自己申请的，泄露后用户需要自己承担损失。所以保护 Key 是最高优先级。

**怎么保护的**：
- 存储：使用 Android 系统的"加密保险箱"（EncryptedSharedPreferences），Key 被 AES-256 加密后存储，即使手机被 root 也无法直接读取
- 传输：仅通过 HTTPS 加密通道发送，不在 URL 中暴露
- 日志：绝对不在日志中打印 API Key 内容

### 2.2 数据加密

**是什么**：用户的对话记录、录音文件、备份数据都需要加密保护，防止被他人获取。

**为什么现在做**：虽然本项目是"纯本地"的，数据都在用户手机上，但手机可能丢失、被他人借用、或被恶意软件扫描。加密是保护用户隐私的最后一道防线。

**目前状态**：
- 录音文件：AES-256-GCM 加密存储（已修复）
- 数据备份：AES-256-GCM 加密 + 密码保护（已修复）
- 对话消息：明文 JSON 存储（待改进）
- 记忆数据库：明文 SQLite 存储（待改进）

### 2.3 空安全

**是什么**：Kotlin 的"空安全"机制要求明确标注变量是否可能为 null。如果未正确处理 null，App 会崩溃。

**为什么重要**：闪退是用户最讨厌的体验。Kotlin 的空安全设计就是为了在编译时发现潜在的崩溃风险。

**本项目处理方式**：
- 优先使用 `?.` 安全调用和 `?:` 默认值
- 避免使用 `!!` 强制非空断言
- 对 `lateinit` 属性使用前检查 `::xxx.isInitialized`

### 3.2 API Key 安全

#### 3.2.1 存储安全（优秀）

[AppConfig.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/AppConfig.kt)

- 使用 `EncryptedSharedPreferences` 存储 API Key
- 主密钥方案：`MasterKey.KeyScheme.AES256_GCM`
- Key 加密：`AES256_SIV`
- Value 加密：`AES256_GCM`
- 初始化失败时抛出 `SecurityException`，**不降级为明文存储**

```kotlin
// AppConfig.kt:L111-L133
private fun getSecurePrefs(context: Context): SharedPreferences {
    return cachedSecurePrefs ?: synchronized(securePrefsLock) {
        cachedSecurePrefs ?: run {
            try {
                val masterKey = MasterKey.Builder(context.applicationContext)
                    .setKeyScheme(MasterKey.KeyScheme.AES256_GCM)
                    .build()
                EncryptedSharedPreferences.create(
                    context.applicationContext,
                    SECURE_PREFS_NAME,
                    masterKey,
                    EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
                    EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
                ).also { cachedSecurePrefs = it }
            } catch (e: Exception) {
                Log.e("AppConfig", "加密存储初始化失败: ${e.message}")
                throw SecurityException("设备不支持加密存储", e)
            }
            cachedSecurePrefs!!
        }
    }
}
```

### 3.3 数据存储安全

#### 3.3.1 录音文件加密（优秀）

[VoiceRecorder.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/speech/VoiceRecorder.kt)

- 录音完成后立即使用 `EncryptedFile` 加密，删除原始明文
- 加密方案：`AES256_GCM_HKDF_4KB`
- 播放时解密到临时文件，使用后由调用方清理
- 24 小时自动清理过期录音文件

```kotlin
// VoiceRecorder.kt:L214-L233
private fun encryptFile(rawFile: File): File? {
    return try {
        val key = getMasterKey(context)
        val encryptedFile = File(rawFile.parentFile, rawFile.name + ENCRYPTED_EXT)
        val encryptedFileWrapper = EncryptedFile.Builder(
            context, encryptedFile, key,
            EncryptedFile.FileEncryptionScheme.AES256_GCM_HKDF_4KB
        ).build()
        rawFile.inputStream().use { input ->
            encryptedFileWrapper.openFileOutput().use { output ->
                input.copyTo(output)
            }
        }
        encryptedFile
    } catch (e: Exception) {
        Log.e(TAG, "加密录音文件失败: ${e.message}", e)
        null
    }
}
```

#### 3.3.2 数据备份加密（优秀）

[DataBackupHelper.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/DataBackupHelper.kt)

- AES-256-GCM 加密备份文件
- PBKDF2WithHmacSHA256 密钥派生（100,000 次迭代）
- 随机 Salt + IV 生成
- 密码仅存储 SHA-256 哈希，不存储明文
- 路径穿越防护（canonicalPath 比对）
- ZIP 炸弹防护（单条目 50MB，累计 200MB）
- 恢复前自动备份当前数据，失败时回滚
