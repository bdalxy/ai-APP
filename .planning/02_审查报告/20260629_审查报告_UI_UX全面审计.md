## 一、严重问题 (P0) — 必须立即修复

### 问题 1: 设置按钮完全不可发现 (根因)

**文件**: `activity_main.xml` L87-L97 + `MainActivity.kt` L353-L360

**现状**:
```xml
<ImageButton
    android:id="@+id/btnSettings"
    android:layout_width="@dimen/icon_size_small"   <!-- 24dp !!! -->
    android:layout_height="@dimen/icon_size_small"   <!-- 24dp !!! -->
    android:src="@drawable/ic_search"                <!-- 搜索图标，不是齿轮!!! -->
    ...
/>
```

```kotlin
// 设置按钮（搜索图标 → 点击搜索，长按设置）
binding.btnSettings.setOnClickListener {
    chatViewModel.toggleSearchMode()   // 点击=搜索，不是设置!
}
binding.btnSettings.setOnLongClickListener {
    // 只有长按才能打开设置
    ActivityTransitionHelper.startWithSlideIn(this, Intent(this, SettingsActivity::class.java))
    true
}
```

**问题分析**:
- 按钮图标是 `ic_search`（搜索图标），不是设置齿轮图标
- 单击打开搜索，只有**长按**才能打开设置页面
- 用户反馈"设置页面消失了"——实际上是设置入口被隐藏在了长按手势里，**99% 的用户不会发现**
- 按钮尺寸仅 24dp x 24dp，远低于 Android 推荐的 48dp 最小触控目标
- 虽然代码注释写明"搜索图标 → 点击搜索，长按设置"，但这是严重的设计缺陷

**修复方案**:
1. 将按钮图标改为 `ic_settings`（齿轮图标）
2. 将单击改为打开设置页面
3. 将搜索功能移到长按，或增加独立的搜索按钮
4. 将按钮尺寸改为至少 48dp

## 二、高优先级问题 (P1)

### 问题 4: 文字颜色对比度不足 (WCAG AA 失败)

**文件**: `colors.xml` L33 + `values-night/colors.xml` L33

| 颜色 | 前景色 | 背景色 | 对比度 | WCAG AA |
|------|--------|--------|--------|---------|
| `text_tertiary` | `#9A8E8D` | `#FDF0F0` (bg_dark) | **~3.0:1** | 失败 |
| `text_secondary` | `#8A7B7A` | `#FDF0F0` (bg_dark) | **~3.7:1** | 失败 (正常文字) |
| `text_tertiary` | `#7A6A8A` | `#1A0E1F` (dark bg) | ~5.6:1 | 通过 |

**问题分析**: 
- `text_tertiary` (#9A8E8D) 用于提示文字、副标题、时间戳，在淡樱粉背景上对比度仅约 3.0:1，低于 WCAG AA 标准的 4.5:1
- `text_secondary` (#8A7B7A) 在淡樱粉背景上对比度约 3.7:1，也不达标
- 这是"文字看不清"的直接原因

**修复方案**:
- `text_secondary` 从 `#8A7B7A` 改为 `#6B5B5A`（对比度提升到 ~5.5:1）
- `text_tertiary` 从 `#9A8E8D` 改为 `#7A6E6D`（对比度提升到 ~4.5:1）
- 深色模式对应调整：`text_secondary` 从 `#B0A0C0` 改为 `#C8B8D8`，`text_tertiary` 从 `#7A6A8A` 改为 `#9A8AAA`

### 问题 5: 部分文字过小，难以阅读

**文件**: 多个布局文件

| 位置 | 文字大小 | 是否合规 |
|------|---------|---------|
| 消息时间戳 (`item_message_ai.xml`) | 11sp | 过小 |
| 编辑标记 "(已编辑)" (`item_message_self.xml`) | 10sp | 过小 |
| 插件面板状态文字 (`bottom_sheet_plugins.xml`) | 10sp | 过小 |
| 会话列表时间 (`item_session_drawer.xml`) | 11sp | 过小 |
| 会话消息数 (`item_session_drawer.xml`) | 11sp | 过小 |
| 设置项副标题 (`SettingsItemSubtitle`) | 13sp | 勉强合规 |
| 按钮文字 (发送) | 18sp | 合规 |
| 消息正文 | 15sp | 合规 |
| 标题 | 17-20sp | 合规 |

**修复方案**: 所有辅助文字（时间戳、状态、标签）从 10-11sp 提升到至少 12sp（`@dimen/text_small`），从 `text_caption`(13sp) 引用。修改 `dimen/text_small` 从 12sp 改为 13sp。

## 三、中优先级问题 (P2)

### 问题 6: 设置详情页内容依赖代码动态填充

**文件**: `activity_settings_detail.xml` L60-L64

**现状**: 详情页内容区域 `detail_content` 是一个空的 LinearLayout，完全依赖 `SettingsDetailActivity.kt` 在代码中动态填充。如果填充逻辑有问题，用户看到的就是空白页。

**风险**: 需要确认 `SettingsDetailActivity.kt` 中的动态填充逻辑是否完整覆盖了所有 9 种设置类型（account, chat, memory, voice, notification, display, about, world_book, data_management）。

---

### 问题 7: 顶栏按钮缺少视觉区分

**文件**: `activity_main.xml` L76-L97

**现状**: 顶栏有两个图标按钮（新对话 + 搜索），但它们都是纯图标，没有任何文字标签，也没有视觉分隔。用户很难区分它们的功能。

**修复方案**: 如果保留两个按钮，应该用分隔线或间距明确分开；建议将搜索功能移到抽屉中，顶栏只保留新对话和设置两个按钮。

---

### 问题 8: 发送按钮非激活态视觉反馈弱

**文件**: `activity_main.xml` L228-L239

**现状**: 发送按钮使用 `AppButton.Primary` 样式，但 `android:enabled="false"` 初始状态。非激活态的视觉反馈（`send_inactive_bg` / `send_inactive_v2_solid`）可能不够明显，用户可能看不出按钮是否可用。

**修复方案**: 非激活态增加 alpha 降低（如 0.4），或使用更明显的灰色/透明色。

---

## 六、修复优先级总结

| 优先级 | 问题 | 修复工作量 | 影响范围 |
|--------|------|-----------|---------|
| **P0** | 设置按钮不可发现（图标+手势） | 小（改1个xml+3行kotlin） | 所有用户 |
| **P0** | 按钮尺寸过小（40dp/24dp） | 小（改2个dimen值） | 所有页面 |
| **P0** | 抽屉缺少设置入口 | 小（加1个TextView） | 设置可达性 |
| **P1** | 文字对比度不足 | 小（改4个颜色值） | 全项目可读性 |
| **P1** | 文字过小（10-11sp） | 小（改1个dimen+多处硬编码） | 全项目可读性 |
| **P2** | 设置详情页内容填充 | 中（需验证代码） | 设置详情页 |
| **P2** | 顶栏按钮视觉区分 | 小 | 主页面 |
| **P2** | 发送按钮非激活态 | 小（加alpha） | 主页面 |
| **P3** | 硬编码文字大小 | 小（改用dimen引用） | 一致性 |

**总计**: 9 个问题，其中 3 个 P0 严重问题，2 个 P1 高优先级问题。**所有 P0/P1 问题修复工作量均很小**（主要是改尺寸值和颜色值），预计 1-2 小时可完成全部修复。
