## 2. API Key 安全

### 2.1 存储安全

**审查文件**: [AppConfig.kt](file:///f:/Trae%20AI/ai-APP/android/app/src/main/java/com/aicompanion/app/AppConfig.kt)

**审查结果**: 优秀

API Key 使用 AndroidX Security Crypto 库的 `EncryptedSharedPreferences` 进行 AES-256 加密存储：

- 主密钥使用 `MasterKey.KeyScheme.AES256_GCM` 生成，存储在 Android Keystore 中
- Key 本身使用 `PrefKeyEncryptionScheme.AES256_SIV` 加密
- Value 使用 `PrefValueEncryptionScheme.AES256_GCM` 加密
- 加密存储初始化失败时抛出 `SecurityException`，**不降级为明文存储**

```kotlin
// AppConfig.kt:L107-L129
private fun getSecurePrefs(context: Context): SharedPreferences {
    return cachedSecurePrefs ?: synchronized(securePrefsLock) {
        cachedSecurePrefs ?: run {
            try {
                val masterKey = MasterKey.Builder(context.applicationContext)
                    .setKeyScheme(MasterKey.KeyScheme.AES256_GCM)
                    .build()
                EncryptedSharedPreferences.create(...)
            } catch (e: Exception) {
                throw SecurityException("设备不支持加密存储，无法安全保存 API Key", e)
            }
        }
    }
}
```

### 2.2 运行时安全

**审查文件**: [deepseek.py](file:///f:/Trae%20AI/ai-APP/android/app/src/main/python/src/api_client/deepseek.py), [_core.py](file:///f:/Trae%20AI/ai-APP/android/app/src/main/python/chat_bridge/_core.py)

**审查结果**: 优秀

- API Key 通过 `Authorization: Bearer <key>` 传递，符合 OAuth 2.0 Bearer Token 规范
- `_ensure_auth()` 实现延迟校验，如果 Key 为空时抛出 `APIKeyError`
- `update_api_key()` 方法支持运行时更新，同步到 session header
- 日志中不记录 API Key 内容（仅记录 "session header 已更新"）
- `set_api_key()` 对空 Key 输入进行校验，拒绝空值

### 2.3 传输安全

- Python 端通过 `requests.Session` 发送，Header 中包含 `Authorization: Bearer <key>`
- 所有请求通过 HTTPS 发送（参见第 4 节网络通信安全）
- API Key 仅通过 HTTPS Header 传输，不出现在 URL 查询参数中

### 结论

- API Key 存储：AES-256 加密，Android Keystore 保护，无降级风险
- API Key 传输：HTTPS only，Bearer Token 方式
- API Key 日志：不记录原始 Key
- 无硬编码 API Key 风险

**无安全问题。**

## 4. 网络通信安全

### 4.1 HTTPS 强制与证书信任

**审查文件**: [network_security_config.xml](file:///f:/Trae%20AI/ai-APP/android/app/src/main/res/xml/network_security_config.xml), [AndroidManifest.xml](file:///f:/Trae%20AI/ai-APP/android/app/src/main/AndroidManifest.xml)

**审查结果**: 优秀

```xml
<!-- network_security_config.xml -->
<base-config cleartextTrafficPermitted="false">
    <trust-anchors>
        <certificates src="system" />
    </trust-anchors>
</base-config>

<domain-config cleartextTrafficPermitted="false">
    <domain includeSubdomains="true">api.deepseek.com</domain>
    <trust-anchors>
        <certificates src="system" />
    </trust-anchors>
</domain-config>
```

- `cleartextTrafficPermitted="false"` 全局禁止明文 HTTP 流量
- 仅信任系统 CA 证书，拒绝用户安装的 CA 证书（防御中间人攻击）
- DeepSeek API 域名使用独立配置，确保仅信任系统证书
- `android:usesCleartextTraffic="false"` 在 AndroidManifest 中双重保险
- Debug 构建允许 user 证书（用于本地调试抓包），Release 构建不受影响

### 4.2 API 请求安全

**审查文件**: [deepseek.py](file:///f:/Trae%20AI/ai-APP/android/app/src/main/python/src/api_client/deepseek.py)

- 使用 `requests.Session` 连接池复用，减少 TLS 握手开销
- 设置连接超时 5 秒和读取超时 30 秒，防止长时间挂起
- 流式请求读取超时加倍（60 秒），确保长文本生成不过早断开
- 所有 API 请求 URL 使用 `https://api.deepseek.com` 硬编码，无法被篡改为 HTTP
- HTTPS 证书验证由 `certifi` 包提供，使用系统信任的 CA 证书链

### 4.3 证书固定评估

当前配置仅信任系统 CA 证书，未实施证书公钥固定（Public Key Pinning）。考虑到：
- DeepSeek API 证书由权威 CA 签发，信任链可靠
- 证书固定存在维护成本（证书更换时需要更新应用）
- 对于非金融类应用，系统 CA 信任已足够安全

**结论**: 当前认证策略合理，无需额外证书固定。

### 结论

- HTTPS 强制通过两层配置确保
- 证书信任链安全
- 无明文 HTTP 通信风险
- 超时配置合理

**无安全问题。**
