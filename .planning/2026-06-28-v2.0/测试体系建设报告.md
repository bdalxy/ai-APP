## 一、概述

本报告记录了 AI Companion v2.0.0 测试体系建设的完整成果。按照项目主控计划，从零开始建立了覆盖 Kotlin 端、Python 端和 UI 层的三级测试体系。

### 1.1 测试框架

| 层级 | 框架 | 用途 |
|:---:|------|------|
| Kotlin 单元测试 | JUnit 5 (Jupiter) + MockK + Robolectric | 纯 JVM 测试，模拟 Android 环境 |
| Kotlin 协程测试 | kotlinx-coroutines-test | 测试 SharedFlow/StateFlow 等协程组件 |
| Python 单元测试 | unittest (标准库) | Python 模块测试 |
| UI 测试 | Espresso + AndroidJUnit4 | 设备/模拟器上的 UI 交互测试 |
| 测试运行器 | Gradle `test` 任务 + `connectedAndroidTest` | 构建系统集成 |

## 七、运行指南

### 7.1 运行 Kotlin 单元测试

```bash
# 在 android/ 目录下执行
cd android

# 运行所有单元测试（JVM）
./gradlew test

# 运行特定测试类
./gradlew test --tests "com.aicompanion.app.MessageTest"

# 运行特定测试方法
./gradlew test --tests "com.aicompanion.app.MessageTest.Defaults.testDefaults"
```

### 7.2 运行 Python 单元测试

```bash
# 在 Python tests 目录下执行
cd android/app/src/main/python/tests

# 运行所有测试
python -m pytest . -v

# 运行特定测试文件
python -m pytest test_world_book.py -v

# 运行特定测试类
python -m pytest test_world_book.py::TestWorldBookMatching -v
```

### 7.3 运行 UI 测试（需要设备/模拟器）

```bash
cd android

# 运行所有 UI 测试
./gradlew connectedAndroidTest

# 运行特定测试类
./gradlew connectedAndroidTest --tests "com.aicompanion.app.MainActivityTest"
```

### 7.4 生成测试覆盖率报告

```bash
cd android

# 在 build.gradle.kts 中添加 JaCoCo 插件后
./gradlew test jacocoTestReport
```

## 八、已知限制与后续建议

### 8.1 当前限制

| 限制 | 说明 | 影响 |
|------|------|------|
| Chaquopy 不可用 | JVM 单元测试无法调用 Python 桥接 | 集成测试仅做接口契约验证 |
| Robolectric 部分限制 | 加密 SharedPreferences 在 Robolectric 中行为不同 | API Key 加密测试需真机验证 |
| 无 CI 集成 | GitHub Actions 未配置 | 无法自动运行测试 |
| 无覆盖率报告 | JaCoCo 未配置 | 无法量化覆盖率 |

### 8.2 后续建议

1. **配置 JaCoCo 覆盖率**：在 build.gradle.kts 中添加 JaCoCo 插件，设定初始目标覆盖率 >= 30%
2. **配置 GitHub Actions CI**：添加 `.github/workflows/test.yml`，在每次 PR 时自动运行测试
3. **真机测试验证**：在真机上运行 `connectedAndroidTest` 验证 UI 测试
4. **补充 ViewModel 测试**：为 ChatViewModel、PluginViewModel 添加单元测试
5. **补充 Python 测试**：为 proactive 模块、plugins 模块添加单元测试
6. **性能测试**：添加内存泄漏检测和性能基准测试
