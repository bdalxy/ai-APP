## 全局前置：UI 主题统一

> **背景**：项目早期定义了"往世乐土玻璃态紫色系"，但仅少数页面有体现，大量页面仍是基础 Android 样式。本次重构要求所有页面统一应用。

**统一规范**：

| 元素 | 规范 |
|------|------|
| 背景 | 深色渐变 `#1A0A2E → #0D0D1A`（`res/drawable/bg_gradient_dark.xml`） |
| 卡片 | 半透明玻璃态 `#1AFFFFFF` + 圆角 16dp + 边框 `#20FFFFFF` |
| 主色 | 紫色系 `#9B59B6`（Primary）/ `#8E44AD`（Dark）/ `#BB6BD9`（Light） |
| 文字 | 主文字 `#FFFFFF` / 次要文字 `#B0B0C0` / 提示文字 `#707088` |
| 按钮 | 紫色填充圆角按钮 / 玻璃态边框按钮 |
| 输入框 | 半透明背景 + 紫色聚焦边框发光 |
| 顶栏 | 半透明玻璃态 + 模糊背景 |

## 任务清单

### 🔴 P0 — Bug 修复（先做，不依赖其他任务）

---

#### TASK-01: 修复 AI 回复为空

- **根因**: `ChatViewModel.kt` 流式 poll `when` 分支缺少 `else` 兜底
- **文件**: `ChatViewModel.kt`, `chat_bridge/__init__.py`
- **操作**:
  1. 在 `callbackFlow` 的 `when` 分支增加 `else` 兜底 → 记录未知状态日志
  2. Python 端 `chat_stream_poll()` 增加详细错误日志
  3. 流式结束时若 `streamingAiMsgIndex` 对应消息仍为空，兜底显示"AI 回复异常，请重试"
- **验收**: 网络异常/Python 崩溃时不再显示空白消息，有明确错误提示
- **预估**: 1h

---

#### TASK-02: 修复语音识别误触发

- **根因**: `wasVoiceInput` 标志位在流式回复中被错误触发
- **文件**: `VoiceController.kt`, `MainActivity.kt`
- **操作**:
  1. `sendMessage()` 入口处（`onNewMessageSent` 回调）确保 `wasVoiceInput` 被重置
  2. `speakAIContentIfNeeded()` 增加 TTS 引擎就绪状态检查，未就绪时静默跳过
- **验收**: 未开启自动朗读时，文字输入发送消息后不弹出语音相关错误提示
- **预估**: 30min
