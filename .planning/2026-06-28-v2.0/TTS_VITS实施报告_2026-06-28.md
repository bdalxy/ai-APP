## 二、已完成的修改

### 2.1 ProGuard 规则修复

**文件**：[proguard-rules.pro](file:///f:/Trae AI/ai-APP/android/app/proguard-rules.pro#L52)

**问题**：Release 构建时 `OfflineTtsVitsModelConfig` 类可能被 R8 混淆，导致运行时 crash。

**修复**：添加 ProGuard keep 规则。

```diff
-keep class com.k2fsa.sherpa.onnx.OfflineTtsMatchaModelConfig { *; }
+keep class com.k2fsa.sherpa.onnx.OfflineTtsMatchaModelConfig { *; }
+keep class com.k2fsa.sherpa.onnx.OfflineTtsVitsModelConfig { *; }
```

### 2.2 VITS 模型下载脚本

**文件**：[download_vits_model.ps1](file:///f:/Trae AI/ai-APP/scripts/download_vits_model.ps1)

**功能**：
- 自动从 Sherpa-ONNX 官方仓库下载 VITS aishell3 模型（~116MB）

### 2.4 SherpaTtsEngine 增强

3. **初始化日志增强**（[L223-L224](file:///f:/Trae AI/ai-APP/android/app/src/main/java/com/aicompanion/app/speech/SherpaTtsEngine.kt#L223-L224)）：

4. **新增 getSpeakerCount() 方法**（[L468-L469](file:///f:/Trae AI/ai-APP/android/app/src/main/java/com/aicompanion/app/speech/SherpaTtsEngine.kt#L468-L469)）：

## 三、用户操作指南

### 3.1 下载 VITS 模型（必须）

```powershell
# 在项目根目录（ai-APP）下运行
cd f:\Trae AI\ai-APP
powershell -ExecutionPolicy Bypass -File scripts/download_vits_model.ps1
```

脚本会自动完成：
1. 下载 `vits-aishell3.tar.bz2`（约 116MB）
2. 解压到 `android/app/src/main/assets/vits-aishell3/`
3. 验证 `model.onnx`、`tokens.txt`、`lexicon.txt` 文件完整性

### 3.2 编译与运行

1. 在 Android Studio 中执行 **Build -> Clean Project**
2. 执行 **Build -> Rebuild Project**
3. 运行 APP 到真机/模拟器

### 3.3 验证 VITS 是否生效

1. 打开 APP，进入「设置 -> 语音设置」
2. 查看「TTS 模型」显示为「自动检测（VITS 优先）」或「VITS (aishell3)」
3. 进入对话，发送一条消息
4. 查看 Logcat 日志：

```bash
adb logcat -s SherpaTtsEngine:D
```

期望日志：
```
SherpaTtsEngine: 使用 VITS (aishell3) 模型...
SherpaTtsEngine: VITS (aishell3) TTS 引擎初始化成功, 采样率=22050, 说话人数=174
SherpaTtsEngine: 开始合成: 你好，我是AI助手...
SherpaTtsEngine: 合成参数: speed=1.0, sid=0
SherpaTtsEngine: 合成完成, samples=xxxxx, sampleRate=22050
SherpaTtsEngine: 播放完成
```

### 3.4 切换说话人

1. 进入「设置 -> 语音设置」
2. 点击「TTS 说话人」
3. 选择不同说话人（0-173）
4. 返回对话，发送消息测试音色变化

### 3.5 切换回 Matcha（回退）

1. 进入「设置 -> 语音设置」
2. 点击「TTS 模型」
3. 选择「Matcha (baker)」
4. 重启 APP 生效
