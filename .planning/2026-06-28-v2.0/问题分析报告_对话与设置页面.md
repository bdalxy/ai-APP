## 一、对话主页面问题

### 1. AI 回复显示为空

**现状分析**：从 [ChatViewModel.kt](file:///f:/Trae AI/ai-APP/android/app/src/main/java/com/aicompanion/app/ChatViewModel.kt#L373-L582) 流式发送逻辑来看，消息流程是：用户消息气泡 → 打字指示器 → 占位 AI 消息 → 流式 token 填充。

**可能原因**：

- Python `chat_stream_start()` 返回了 JSON 错误（被当作 streamId 解析），`streamId.startsWith("{")` 检查后显示错误信息，但错误信息可能不明确
- `chat_stream_poll()` 返回的 status 不是 "streaming"/"done"/"error" 之一，`when` 分支未覆盖，消息未被更新
- 网络异常导致 DeepSeek API 调用失败，错误信息未正确传递到 UI
- 流式协程被意外取消，`finally` 块中 `isStreaming = false` 但 AI 消息已为空白

**根因定位**：[ChatViewModel.kt:552](file:///f:/Trae AI/ai-APP/android/app/src/main/java/com/aicompanion/app/ChatViewModel.kt#L552) 的 `callbackFlow` 的 `when` 分支缺少 `else` 兜底，当 poll 返回未知状态时静默丢弃。

**建议方案**：

- 在 `when` 分支增加 `else` 兜底，记录未知状态日志
- Python 端 `chat_bridge` 增加更详细的错误日志
- 在 `streamingAiMsgIndex` 消息未被更新时，兜底显示"AI 回复异常，请重试"

**归属**：Bug 修复（v2.0 品质打磨阶段 1）

### 2. 自动朗读未开启却弹出语音识别失败提示

**现状分析**：[MainActivity.kt:240-244](file:///f:/Trae AI/ai-APP/android/app/src/main/java/com/aicompanion/app/MainActivity.kt#L240-L244)：

```kotlin
if (!AppConfig.getAutoReadAloud(this@MainActivity)) {
    voiceController.speakAIContentIfNeeded(fullContent)
}
```

在 [VoiceController.kt:344-352](file:///f:/Trae AI/ai-APP/android/app/src/main/java/com/aicompanion/app/VoiceController.kt#L344-L352) 的 `speakAIContentIfNeeded` 中，即使 `autoRead=false`，如果 `wasVoiceInput=true`（用户通过语音输入），也会触发 TTS 朗读。如果 TTS 引擎初始化失败，就会弹出错误。

**根本原因**：`wasVoiceInput` 标志位在语音输入后的流式回复中被错误触发，导致非语音输入场景下也走了 TTS 路径。

**建议方案**：

- 在 `sendMessage()` 入口处（`onNewMessageSent` 回调）确保 `wasVoiceInput` 被重置
- `speakAIContentIfNeeded` 中增加 TTS 引擎就绪状态检查，未就绪时静默跳过而非弹错误

**归属**：Bug 修复（v2.0 品质打磨阶段 1）

### 3. 新对话按钮不顺畅 + 切换会话 UI 粗糙

**现状分析**：

**参考 tavo 的设计**：tavo 通过左上角"三"（DrawerLayout 抽屉）承载会话列表和导航，点击即从左侧滑出，列表项包含角色头像、名称、最后消息预览。

  - 会话名称
  - 最后消息预览（截断 30 字）
  - 相对时间戳（"3 分钟前"）
  - 消息数量
- 支持滑动删除会话
- 抽屉内底部增加"新建会话"按钮

**归属**：UI 体验优化（v2.0 品质打磨阶段 2）

### 3. "记忆设置"参数粗糙 + 记忆管理不应在此 【已决策】

**现状分析**：[SettingsDetailActivity.kt:332-358](file:///f:/Trae AI/ai-APP/android/app/src/main/java/com/aicompanion/app/SettingsDetailActivity.kt#L332-L358)：

**用户决策**：参考 tavo 的双抽屉 + UI 布局，结合项目主题（往世乐土深色玻璃态）给出自己的特色。

- 记忆设置页面只保留参数配置，每个参数增加说明文案

- UI 风格：玻璃态卡片 + 紫色系主题色

**归属**：设置页面重构 + 功能入口调整 + 对话主页面抽屉整合
