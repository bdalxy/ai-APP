# 独立 Android APP 可行性分析报告 (v3.0)

**项目代号**: AiCompanion-Native
**版本**: v3.0
**日期**: 2026-05-27
**分析方式**: 多智能体并行分析 + 交叉验证

---

## 新约束

1. **完全自建 Android APP**，不依赖微信/Telegram/任何第三方 IM
2. **AI 推理**: DeepSeek API（云端），不在本地跑 LLM
3. **语音声线**: PC 端 GPT-SoVITS 训练 → ONNX 导出 → APP 推理
4. **声线混合**: 预设声线按比例混合（Speaker Embedding 线性插值）
5. **单用户**: 自己一个人用，无需多用户隔离
6. **主动消息**: 本地通知 + WorkManager

## 2. 声线方案详细分析

### PC 训练 → 手机推理流程

```
PC 端（一次性操作）:
  ① 收集参考音频（10-30秒）
  ② GPT-SoVITS 微调训练
  ③ 导出 ONNX 模型（~80MB/声线）
  ④ 传输到手机（USB/局域网）

手机端:
  ⑤ APP 加载 ONNX (ONNX Runtime Mobile)
  ⑥ 用户输入文字 → TTS 推理 → 播放
```

### 声线混合原理

```
声线A的 embedding: [e₁, e₂, ..., e₅₁₂]
声线B的 embedding: [f₁, f₂, ..., f₅₁₂]

70% A + 30% B:
E_mix = 0.7×[e₁...e₅₁₂] + 0.3×[f₁...f₅₁₂]

将 E_mix 注入 TTS 解码器 → 混合声音
```

**风险**: 相近声线（两个女声）效果较好；差异大的（男+女）可能不自然。

## 3. 设备要求

| 项 | 要求 | 说明 |
|----|------|------|
| Android 版本 | 10+ (API 29) | 95% 设备覆盖 |
| RAM | 4GB+ | 不需要本地 LLM，TTS 仅需 ~500MB |
| 存储 | 200MB APK + 100MB/声线 | 可扩展 |
| 网络 | 需要联网 | DeepSeek API 调用 |

> **关键变化**: 旧方案需要 12GB+ RAM 旗舰机，新方案 4GB 中端机即可。原因是**不再本地跑 LLM**。
