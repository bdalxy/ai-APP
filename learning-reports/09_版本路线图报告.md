### 3.4 可降级功能与砍功能红线

> **MVP 原则**：如果关键功能在 Week 3 验证不通过，必须有 Plan B，不能因此阻塞发版。

| 功能 | 默认方案 | Plan B（降级方案） | 触发条件 | 降级影响 |
|------|----------|-------------------|----------|----------|
| **V1.0-14 语音+流式深度集成** | Sherpa-ONNX 离线 TTS + 流式联动 | **仅文本对话发布**，语音功能标注"实验性" | 真机 TTS 初始化失败率 > 20% 或流式联动不稳定 | 丧失语音交互差异化，但核心对话闭环不受影响 |
| **V1.0-13 ContextBuilder 接入** | 分层注入（核心+扩展+背景） | **仅注入核心记忆**，跳过扩展和背景层 | Token 预算管理异常导致截断 | 记忆注入质量下降，但不影响基本对话 |
| **V1.0-02 RenderScript 迁移** | RenderEffect（API 31+） | **半透明遮罩降级方案**（无模糊效果） | 低版本 Android 兼容性问题 | 视觉效果降级，功能不受影响 |

> **砍功能红线**：以下情况必须延迟发版，不可降级绕过：
> - 崩溃日志含用户明文（V1.0-01）— 隐私红线
> - 记忆合并事务保护缺失（V1.0-06）— 数据完整性红线
> - Python 调用无超时保护（V1.0-10）— ANR 红线
> - 真机验证 5 大模块任一未通过（V1.0-18）— 质量红线

### 6.4 缺失维度补全（本次修订新增）

> 原始路线图存在三个"隐形维度"缺失，现逐一补全：

#### 6.4.1 自动化测试覆盖

| 测试层级 | 工具 | 覆盖目标 | 插入节点 | 预估工时 |
|----------|------|----------|----------|----------|
| **Python 单元测试** | pytest + unittest.mock | 记忆系统核心链路（extractor/retriever/consolidator）≥ 60% | v1.0 Week 2（与核心闭环并行） | 4h |
| **Kotlin 单元测试** | JUnit 5 + Mockito | ChatViewModel / AppConfig / PluginRegistry | v1.1 Week 4（与 VM 重构并行） | 4h |
| **UI 自动化测试** | Espresso | 核心页面（聊天/设置/记忆管理）冒烟用例 | v1.1 Week 6（与回归测试并行） | 3h |
| **性能基准测试** | JMH / 手动计时 | 1000 条记忆检索 < 500ms | v1.0 Week 3（真机验证时并行） | 2h |

> **关键原则**：自动化测试与开发并行，不占额外开发工时。v1.1 的 ChatViewModel 重构（V1.1-01）若无自动化回归保护，6h 工时根本兜不住重构引入的 Bug。

#### 6.4.2 数据迁移方案

| 版本 | Schema 变更 | 迁移策略 | 回滚方案 |
|------|------------|----------|----------|
| **v1.0→v1.1** | 记忆系统：consolidator + lifecycle 合并 | **渐进式迁移**：新增字段设默认值，旧字段保留一个版本周期，SQLite `ALTER TABLE` 不删列 | 保留 `.db.before_migrate` 备份，检测到崩溃自动回滚 |
| **v1.1→v1.2** | MemoryCategory 简化 | **映射迁移**：旧 Category 枚举值 → 新简化值，写入迁移日志 | 迁移前全量 JSON 导出，失败时从 JSON 恢复 |
| **通用策略** | 所有版本 | `DB_VERSION` 递增 + `onUpgrade()` 分支处理，首次启动显示"数据升级中"进度 | 每次升级前自动创建 `.db.backup_v{X}` 备份 |

> **红线**：任何数据迁移必须有回滚方案，禁止直接 `DROP COLUMN` 或不可逆操作。

#### 6.4.3 Android 版本兼容基线

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **minSdk** | **26 (Android 8.0)** | 覆盖 95%+ 活跃设备，Chaquopy Python ABI 兼容 |
| **targetSdk** | **34 (Android 14)** | 当前最新稳定版 |
| **compileSdk** | 34 | 与 targetSdk 一致 |
| **真机验证覆盖** | **Android 10 / 12 / 14** 三台设备 | 低/中/高三个 API 级别各一台 |
| **已知风险 API** | RenderEffect 需 API 31+，Sherpa-ONNX 需 ARM64 ABI | 低版本降级方案已就绪 |

> **真机验证 6h 风险警告**：若只测 Android 14 一台设备，发布后可能在 Android 10 设备上大面积闪退。必须覆盖至少 3 个 API 级别的真机。
