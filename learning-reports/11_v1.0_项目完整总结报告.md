## 一、项目概述

**AI Companion**（AI 伙伴）是一款运行在 Android 设备上的 AI 角色扮演聊天应用。

| 项目 | 说明 |
|------|------|
| 类型 | 独立 Android 原生 APP |
| 技术栈 | **Kotlin 2.0.20** + **Chaquopy Python 3.10** + **DeepSeek API**（deepseek-v4-flash） |
| 设计主题 | 樱羽主题 v1.0 — 淡樱粉 / 淡粉蓝 / 淡天蓝 / 天使白，半透明毛玻璃玻璃态 |
| 主色调 | 淡天蓝 `#B0C4DE`（主色），淡樱粉 `#FFDCE3`（辅色） |
| 设备兼容 | Android 8.0（API 26）至 Android 14（API 34），仅 ARM64 架构 |
| 构建系统 | Gradle 8.x + AGP |

### 核心理念

AI Companion 不只是聊天机器人。它拥有**长期记忆**，能记住你们聊过的每一件事；它理解**世界观**，能扮演不同角色活在设定好的世界里；它还能**主动关心你**，在合适的时间发来问候。

## 二、核心功能清单

### 2.1 对话系统

| 功能 | 说明 | 技术亮点 |
|------|------|----------|
| 流式对话 | AI 回复逐字显示，打字机体验 | callbackFlow 流式架构 |
| 角色扮演 | 支持自定义角色人设，AI 按照设定性格回复 | 系统提示词注入 + 角色卡管理 |
| 多消息拆分 | 长消息自动拆分为多条发送，避免截断 | 智能分段算法 |
| 对话搜索 | 关键词搜索历史对话 | 实时过滤 |
| 对话导出 | 导出为 JSON / TXT 格式 | ChatExporter 模块 |
| 重试机制 | 单条消息重试，重新生成 AI 回复 | 失败自动重试 |
| 消息拆分 | 编辑模式拆分消息 | 长按菜单操作 |

**对话数据流：**

```
用户输入 → MainActivity → ChatViewModel
  → Python chat_bridge.chat_stream_start()
    → RolePlayer.chat()
      → 检索记忆 → 匹配世界书 → 构建 Prompt → DeepSeek API
    → 流式返回 token 队列
  → ChatViewModel 轮询取 token
  → ChatAdapter 逐字更新 UI
  → 对话结束后自动提取记忆 → 存入 SQLite 向量库
```

### 2.2 记忆系统（⭐ 核心亮点）

本项目的记忆系统是一套完整的认知架构，包含 18 个 Python 模块，从"记住一句话"到"理解一段关系"全链路覆盖。

| 模块 | 功能 |
|------|------|
| **提取器（Extractor）** | 规则 + LLM 双引擎提取，11 细粒度分类，自动过滤提问句 |
| **向量存储（VectorStore）** | SQLite 持久化 + 加密 + 倒排索引，支持余弦相似度搜索 |
| **检索器（Retriever）** | 向量 / BM25 / 图谱 / 规则 4 策略混合检索 |
| **知识图谱（KnowledgeGraph）** | 实体提取 + 关系提取 + 图谱遍历，构建记忆之间的关联 |
| **摘要器（Summarizer）** | 单轮 / 会话 / 主题三层摘要，渐进式压缩 |
| **上下文构建器（ContextBuilder）** | 多策略检索 + Token 预算管理，智能注入对话上下文 |
| **合并器（Consolidator）** | 记忆合并 + 冲突解决 + 重要性重评估 |
| **衰减器（Decay）** | 时间衰减 + 访问提振，模拟人类记忆遗忘曲线 |
| **生命周期（Lifecycle）** | 重要性校准 + 过期清理 + 归档触发 |
| **归档器（Archiver）** | 长期记忆归档，冷热分离 |
| **分析器（Analyzer）** | 趋势分析 + 用户画像，理解用户偏好 |
| **缓存（MemoryCache）** | 多级缓存 LRU + TTL，加速热点记忆访问 |
| **备份（Backup）** | 备份与恢复，数据安全 |

**性能基准测试数据：**

| 指标 | 数值 |
|------|------|
| 1000 条记忆插入 | **12,210 条/秒** |
| 向量搜索（1000 条） | **0.040 秒** |
| 多策略检索（1000 条） | **0.088 秒** |

|------|------|

| 关键词匹配 | 精确匹配 + 模糊匹配 |
| 多级排序 | 匹配类型 → 优先级 → 时间戳，智能排序 |

| 概率注入 | 可选择按概率随机注入，增加多样性 |

## 三、技术架构

### 3.1 分层架构

```
┌─────────────────────────────────────────────────────────┐
│                    Kotlin UI 层（Android）                │
│  MainActivity / ChatViewModel / ChatAdapter / 13 个页面  │
├─────────────────────────────────────────────────────────┤
│                  Chaquopy 桥接层（Python ↔ Kotlin）       │
│  chat_bridge/ — 60+ 导出函数，双向调用                   │
├─────────────────────────────────────────────────────────┤
│                    Python 引擎层                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐  │
│  │ 对话引擎  │ │ 记忆系统  │ │ 世界书   │ │ 插件系统  │  │
│  │chat_engine│ │ memory/  │ │world_book│ │ plugins/ │  │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘  │
├─────────────────────────────────────────────────────────┤
│                    外部 API 层                           │
│         DeepSeek API（deepseek-v4-flash）               │
│         wttr.in（天气）                                  │
└─────────────────────────────────────────────────────────┘
```

### 3.2 记忆系统架构

```
Orchestrator（编排器）
  ├── Extractor（提取器）    → 从对话中提取记忆
  ├── VectorStore（向量库）  → SQLite 持久化存储
  ├── Retriever（检索器）    → 4 策略混合检索
  │     ├── 向量相似度
  │     ├── BM25 关键词
  │     ├── 知识图谱遍历
  │     └── 规则匹配
  ├── KnowledgeGraph（知识图谱）→ 实体+关系
  ├── Summarizer（摘要器）   → 三层摘要
  ├── Consolidator（合并器） → 去重+合并
  ├── Decay（衰减器）        → 时间衰减
  └── Lifecycle（生命周期）  → 归档+清理
```

## 七、已知局限

### 7.1 功能局限

| 局限 | 详情 | 影响 |
|------|------|------|
| 笑话插件为模拟数据 | 内置 20 条笑话，无真实 API | 低，仅娱乐功能 |
| 提醒插件不持久化 | 使用 threading.Timer，杀进程丢失 | 中，提醒功能不可靠 |
| 世界书运行时 CRUD 缺失 | 无法在运行时动态增删改条目 | 低，需重启应用 |
| 消息编辑功能缺失 | 无法编辑已发送的消息 | 低，不影响核心对话 |
| 角色头像选择未实现 | 字段存在但无图片选择功能 | 低，仅视觉 |
| 记忆参数设置为占位 | 容量/去重/衰减参数不可调 | 低，默认值已优化 |

### 7.2 平台局限

| 局限 | 详情 |
|------|------|
| 无云端同步 | 数据仅存储在本地，卸载即丢失（已支持本地备份/恢复） |
| 仅 Android 平台 | 不支持 iOS / 桌面 / Web |
| 仅 ARM64 架构 | 不支持 x86 模拟器 |
| 无 Kotlin 单元测试 | 仅 Python 端有 77 个测试 |
| 无 CI/CD 自动构建 | 需手动构建 Release APK |
