## 三、核心数据流

### 3.1 完整对话流程

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 用户输入文字 → MainActivity.sendMessage()                │
│    文件: MainActivity.kt                                    │
│    └─→ ChatViewModel.sendMessage()                         │
│        文件: ChatViewModel.kt                               │
│        └─→ lifecycleScope.launch(IO) {                     │
│              pythonModule.callAttr("chat_stream_start", msg)│
│            }                                                │
├─────────────────────────────────────────────────────────────┤
│ 2. Python 端处理                                            │
│    文件: chat_bridge/_core.py                               │
│    └─→ _state._executor.submit(stream_worker)  # 后台线程   │
│        └─→ _ctx.role_player.chat(msg)                      │
│            文件: src/chat_engine/role_player.py             │
│            ├─→ 注入记忆 (memory.orchestrator.recall)        │
│            ├─→ 匹配世界书 (world_book.match)                │
│            ├─→ 构建 System Prompt (prompt_builder.build)    │
│            ├─→ 调用 DeepSeek API (deepseek.chat_stream)     │
│            │   文件: src/api_client/deepseek.py             │
│            └─→ token 逐个写入 queue.Queue                   │
├─────────────────────────────────────────────────────────────┤
│ 3. Kotlin 端轮询取 token                                    │
│    ChatViewModel 每30ms调用:                                │
│    pythonModule.callAttr("chat_stream_poll", streamId)      │
│    └─→ 从队列取 token → 更新 UI                             │
│    └─→ ChatAdapter.notifyItemChanged() 逐字显示             │
├─────────────────────────────────────────────────────────────┤
│ 4. 对话结束后台处理                                          │
│    └─→ 记忆提取: orchestrator.remember()                   │
│        文件: src/memory/orchestrator.py                     │
│        ├─→ 规则模式提取 (extractor)                          │
│        ├─→ LLM模式提取 (每5轮)                              │
│        ├─→ 向量化存储 (vector_store)                        │
│        └─→ 缓存失效 + 自动备份检查                           │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 关键桥接调用

所有 Kotlin → Python 调用都通过 Chaquopy：

```kotlin
// Kotlin 端
val python = Python.getInstance()
val module = python.getModule("chat_bridge")

// 设置 API Key
module.callAttr("set_api_key", apiKey)

// 初始化聊天引擎
module.callAttr("init", tokenPreset)

// 初始化记忆系统
module.callAttr("init_memory", dbPath)

// 流式对话
val streamId = module.callAttr("chat_stream_start", userMessage).toInt()
// 轮询取 token
val result = module.callAttr("chat_stream_poll", streamId).toString()
// 取消
module.callAttr("chat_stream_cancel", streamId)
```

## 五、关键设计决策

| 决策 | 原因 |
|------|------|
| **Chaquopy 直调** | 不经过序列化，直接调用 Python 函数，性能最优 |
| **队列+轮询流式** | Chaquopy 无法迭代 Python 生成器，用队列中转 |
| **SQLite 自建向量存储** | 避免引入 numpy/heavy 依赖，适配 Android 环境 |
| **懒加载子模块** | 记忆系统 6 个子组件按需初始化，减少启动开销 |
| **EncryptedSharedPreferences** | AES-256 加密存储 API Key |
| **手动打包 Python 依赖** | 避免 Chaquopy pip 在线安装的兼容问题 |
| **arm64-v8a only** | 仅支持真机，减少 APK 体积和兼容问题 |
